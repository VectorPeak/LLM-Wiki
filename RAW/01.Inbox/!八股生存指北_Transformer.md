Transformer 模型由 Google 于 2017 年提出，其核心架构完全基于注意力机制。该模型主要由以下五个部分组成。

编码器由 N 个（通常取 6）相同层堆叠而成。每个编码器层包含多头自注意力机制和前馈神经网络两个子模块。其中，多头自注意力机制使每个 token 能够关注输入序列中的所有其他 token，从而捕获全局依赖关系；前馈神经网络则对每个位置的表示进行非线性变换。这两个子模块均采用残差连接与层归一化，以提升训练稳定性。

解码器同样由 N 个相同层堆叠而成，用于自回归生成输出序列。每个解码器层包含三个子模块：掩码多头自注意力机制、多头交叉注意力机制以及前馈神经网络。掩码多头自注意力通过因果掩码防止当前位置访问未来信息；多头交叉注意力中，查询向量来自解码器，键向量与值向量来自编码器，从而实现对输入序列的动态聚焦；前馈神经网络的结构与编码器中的相同。解码器的各子模块也使用残差连接和层归一化。

位置编码用于弥补模型本身无序性的缺陷，将位置信息注入 token 嵌入中。原始 Transformer 采用正弦或余弦函数生成固定编码，后续变体则使用了可学习的位置编码或旋转位置编码（如 RoPE）。

输入输出嵌入层负责将离散的 token ID 映射为连续向量（维度为 d\_model）。编码器与解码器通常共享词嵌入矩阵，有时也与输出层的权重进行共享（即 weight tying）。

输出层将解码器的最终输出经线性投影后接 Softmax 函数，生成词汇表上的概率分布，用于预测下一个 token。

需要注意的是，不同应用场景可能仅使用上述结构中的一部分。例如，GPT 仅使用解码器部分，BERT 仅使用编码器部分，而机器翻译等序列到序列任务则使用完整的编码器-解码器架构。
