# RL-强化学习(Reinforcement Learning)
#### 什么是RL?
强化学习是一种让智能体在环境中不断通过 **试错 + 奖励反馈** 来学习决策的方法

它不像监督学习那样直接给标准答案，而是让模型在环境中不断尝试动作，根据结果获得奖励或惩罚，逐渐学会：在什么状态下，采取什么动作，才能获得更大的长期收益

| 强化学习核心要素       | 含义          | 例子             |
| -------------- | ----------- | -------------- |
| Agent 智能体      | 做决策、学习的人或模型 | 玩游戏的 AI、小狗、机器人 |
| Environment 环境 | 智能体互动的外部世界  | 游戏界面、客厅、道路     |
| State 状态       | 当前情况        | 蛇的位置、食物位置、车辆速度 |
| Action 动作      | 智能体可以做的选择   | 向左、向右、加速、刹车    |
| Reward 奖励      | 环境给出的反馈信号   | 得分 +1，撞墙 -10   |
| Policy 策略      | 智能体学到的行动方案  | 看到食物在右边，就向右走   |

**举例辅助理解**
如果我们想要训练一个玩贪吃蛇的AI
- **Agent 智能体**：控制贪吃蛇的那个AI程序
- **Environment 环境**：贪吃蛇的游戏界面
- **State 状态**：当前游戏画面的所有信息（蛇头的位置、食物的位置、身体的位置等）
- **Action动作**：AI可以执行的操作：向上、向下、向左、向右
- **Reward 奖励**：
	- 吃到食物：+10分（巨大正奖励）
	- 撞到墙或自己的身体：游戏结束（巨大负奖励）
	- 什么都没发生，只是移动了一步：-0.1分（微小负奖励），这是为了鼓励AI尽快找到食物，而不是无所事事地转圈

AI一开始会像无头苍蝇一样乱撞，很快就死了。但通过几万、几十万局的试错，它会慢慢学到：
- “在食物在我右上方”这个状态下，选择向右或向上的动作，最终吃到食物的概率更大（长期奖励更高）
- “在我的身体堵住了左边路口”这个状态下，选择向左的动作会导致立刻死亡（巨大负奖励），这个动作绝对要避免
最终，这个AI会成长为一个永不犯错、能吃到满屏食物的贪吃蛇大师


#### RL核心原理
强化学习最终要做的是更新策略，让高价值动作更容易被选择，低价值动作更少被选择

如果某个动作 $a$ 在状态 $s$ 下带来了更高回报，那么就提高它的概率：
$$\pi_{\text{new}}(a \mid s) > \pi_{\text{old}}(a \mid s)$$
如果某个动作带来了较低回报或惩罚，那么就降低它的概率：
$$\pi_{\text{new}}(a \mid s) < \pi_{\text{old}}(a \mid s)$$
更一般地，策略优化可以表示为：
$$\theta^* = \arg\max_{\theta} \mathbb{E}_{\pi_\theta} \left[ \sum_{t=0}^{T} \gamma^t r_t \right]$$

| 符号            | 含义                                 |
| ------------- | ---------------------------------- |
| $θ$           | 模型/策略的参数。比如神经网络里的权重。               |
| $θ*$          | 最优参数。也就是训练后希望得到的那组最好参数。            |
| $arg max_θ$   | 在所有可能的 θ 里面，找出能让后面那个目标最大的 θ。       |
| $π_θ$         | 由参数 θ 控制的策略。也就是“在某个状态下，模型会怎么选择动作”。 |
| $E_{π_θ}[·]$  | 期望值。因为模型的动作可能有随机性，所以要看平均表现。        |
| $Σ_{t=0}^{T}$ | 从第 0 步到第 T 步，把每一步奖励加起来。            |
| $r_t$         | 第 t 步得到的奖励。                        |
| $γ$           | 折扣因子，通常 0 ≤ γ ≤ 1。                 |
| $γ^t r_t$     | 第 t 步奖励的折扣值。越远的未来奖励，权重可能越小。        |
优化目标是训练模型参数 θ，让它形成一个策略 π_θ，并让这个策略在长期平均意义下拿到最多奖励

# RLHF-基于人类反馈的强化学习(Reinforcement Learning from Human Feedback)
#### 什么是RLHF?
RLHF（Reinforcement Learning from Human Feedback，人类反馈强化学习）起到的作用是，通过将人类的反馈纳入训练过程，为机器提供了一种自然的、人性化的互动学习过程

RLHF 就是：先让人类当“裁判”，教一个“打分机器”学会判断什么是好回答；然后再让这个“打分机器”去当教练，训练一个“聊天机器人”学会说人话、说好话。它本质上是强化学习（RL）的一种高级玩法，关键是把人类的喜好（Human Feedback）加了进去


**为什么需要 RLHF？直接教不行吗？**
因为“好”这个概念太主观、太复杂了
你没法写规则说：“每句话必须包含一个比喻，情绪要激动，音量提高20%”
但你可以让人类用比较的方式说：“我觉得这个比那个好”

**RLHF 就是把这种人类的直觉和偏好，转化成 AI 能理解的“数字奖励”，然后让 AI 自己去学**



**RLHF基础流程**
先训练一个基础模型（SFT）
- 类似小孩学会基础知识，比如会说话、写字

训练奖励模型（Reward Model）
- 人类给模型的输出评分，高分表示好、低分表示不好
- 奖励模型就是把人类喜好变成可以计算的“奖励分数”

用奖励模型优化模型（强化学习阶段）
- 模型尝试不同输出 → 奖励模型评分 → 模型学习选择高分输出
- 最终模型能生成“人类更满意”的答案

01、什么是PPO?

PPO全称为Proximal Policy Optimization（近端策略优化）。该算法由OpenAI于2017年提出，PPO是强化学习里面常用的算法。它其实就是强化学习中的一种“聪明且稳妥的试错方法”。

PPO 的核心就是：在尝试新动作时，要“小步慢走”，别一下子玩得太猛，避免从一个错误直接掉进另一个更大的错误里。

目标：学做一道新菜（糖醋排骨）

假设你是一个厨师（智能体），你想学会做出一道完美的糖醋排骨（目标）。你的朋友是美食家，会给你打分（奖励模型）。

第一种方法：野蛮试错（旧的强化学习方法）
1.第一次做：你完全凭感觉，醋放了5勺。朋友一尝，酸得脸都皱了，说：“太酸了！0分！”（负奖励）。
2.第二次做：你想，上次太酸了，这次我彻底改变策略，一勺醋都不放！结果朋友说：“这成了红烧排骨了，不是糖醋味，不及格！”（又是负奖励）

问题在哪？

你的学习过程像坐过山车，策略变化太剧烈了。从“狂放醋”极端地跳到“完全不放醋”，虽然你本意是想改进，但结果可能一次比一次差，浪费了大量食材（数据），学习效率很低，而且很不稳定。

### 02、举例什么是PPO?

第二种方法：PPO式学菜（聪明稳妥的试错）

现在，你用PPO的思想来学。

1.建立基准：你第一次做，还是放了5勺醋，得了0分。但这次PPO告诉你：“别慌，我们以这次的做法为‘基准线’”
2.小幅度调整：PPO不会允许你下次直接改成放0勺醋。它会说：“变化太大了！风险不可控！我们这次只做一点点微调，比如，试试放4勺醋或者4.5勺醋怎么样？”
3.评估结果：
  。如果你放4.5勺醋，朋友打60分（比0分有进步），PPO就会说：“嗯！这个微调方向是对的！下次可以继续沿着‘稍微减酸’这个方向探索，但幅度还是要小”
  。如果你放4勺醋，朋友说“酸味不够”，只打了30分（比基准的0分好，但比4.5勺的60分差），PPO会说：“看来减酸减得有点多，效果不是最佳。我们得保守一点，不要完全采纳这个改动”

4.核心限制：PPO有一个“信任区域”的概念。就像你给学习过程加了一个“安全绳”，规定每次尝试的新配方，不能和旧配方相差太远。它阻止你进行那些“步子迈得太大”的危险尝试。（比如直接从5勺醋跳到0勺）

结果就是：你通过一系列“小步慢走”的稳健尝试：5勺 -> 4.5勺 -> 4.8勺 -> 4.3勺 ... 最终稳稳地找到了那个“酸甜黄金比例”，比如4.2勺醋。整个过程没有出现灾难性的失败，效率更高，学习曲线更平滑

03、PPO总结

PPO就像是一个严谨的教练或导航系统：

- 目标：带你到达目的地（最大化奖励）
- 方法：允许你尝试新的路线（探索），但绝不会让你一下子从高速公路上冲进旁边的小河里（限制更新幅度）
- 好处：学习过程非常稳定、高效，不容易“学崩”，这是它成为当今强化学习主流算法的根本原因

# DPO-
#### 什么是DPO?

01、什么是DPO？

DPO（Direct Preference Optimization）是强化学习领域的一种偏好优化算法，主要用于从人类偏好数据中直接学习策略，而非依赖传统的奖励函数。DPO 的出现，是因为RLHF这个过程“太绕、太麻烦”了。它直接跳过了“训练奖励模型”这个中间商，让AI通过直接比较“好答案”和“坏答案”来学习，一步到位。

先回顾RLHF的“麻烦”：开一家甜品鉴定学校
想象一下，你想训练一个AI成为甜品大师。
RLHF的做法（三部曲）：
第一步：示范 你教它基础烘焙（监督微调）。
第二步：培养鉴定官 你让它做很多甜品，你亲自品尝并给它们排序（比如提拉米比芝士蛋糕好，芝士蛋糕比马卡龙好）。然后用这个排序数据，专门训练一个“甜品鉴定官”（奖励模型）。这个鉴定官学会了你的口味。
第三步：自我修炼 让AI自己疯狂做甜品，每次做完都请“甜品鉴定官”来打分，AI根据分数慢慢调整配方。

你觉得麻烦在哪？
成本高：你需要训练和维持一个独立的“鉴定官”（奖励模型），这需要额外的数据和算力。
可能出错：如果“鉴定官”本身没学好，或者口味有偏差，那AI在第三步就会朝着错误的方向“修炼”，所谓“垃圾进，垃圾出”。
过程复杂：就像你想教人做菜，却非得先培养一个美食评论家当传话筒。


02、什么是DPO?

DPO的巧妙解决方案：直接“品味对决”

DPO觉得上面那套流程太繁琐了。它说：“我们何必专门养一个鉴定官呢？让AI自己尝一下，直接知道哪个好哪个坏不就行了吗？”

DPO的流程（一步到位）：

1.收集对比数据：你（人类）直接准备好一堆对决数据。对于同一个问题或指令，你提供一个“优选答案”（比如一个专业厨师写的完美菜谱）和一个“劣选答案”（比如一个新手写的漏洞百出的菜谱）。
2.直接告诉AI：你直接对AI说：“看，这道题，这个是好答案，那个是差答案。你以后就要学着生成像好答案那样的，避免生成像差答案那样的。”
3.AI的直接领悟：DPO通过一种巧妙的数学方法，直接调整AI模型本身的参数，使得模型生成“好答案”的概率远远大于生成“差答案”的概率。它不需要中间那个“奖励模型”来打分，它自己内心就形成了对“好”与“坏”的判断。

核心妙处：DPO把“训练奖励模型”和“用奖励模型优化AI”这两个步骤，合并成了一个简单的“分类”问题。它的目标函数直接就是：“请增大好答案的概率，减小坏答案的概率。”

# GRPO-
#### 什么是GRPO?

01、什么是GRPO?

GRPO（Group Relative Policy Optimization，群体相对策略优化）是一种针对大语言模型的强化学习微调算法。它通过对同一问题生成一组（G个）不同回答，并进行组内比较来计算相对优势，从而优化模型策略，无需训练额外的价值模型。

GRPO，中文叫“群体相关策略优化”。你可以把它想象成一位特别会教学生的“超级老师”，用来训练AI模型（比如ChatGPT这样的聊天机器人）。

这个“超级老师”的教学方法很特别：

1.布置一道题（比如“解释一下为什么天空是蓝色的”）。
2.让AI模型用不同的思路一次性写出多个答案（比如G=8个答案）。
3.“老师”不是简单地判断对错，而是把这8个答案放在一起比较，评出个“优、良、中、差”。
4.最后告诉AI：“你写的那个比平均分高的答案，里面的用词和思路很好，要多用；那个垫底的答案，里面的说法有问题，要避免。”

生活化例子：学做“西红柿炒鸡蛋”

假设我们要训练一个“AI厨师”学会做美味的西红柿炒鸡蛋。

1. 用PPO方法（传统一对一老师）：
•第1次：AI厨师做了一盘菜。你（奖励模型）尝了后说：“咸了，60分。”AI调整：下次少放点盐。
•第2次：AI又做了一盘。你尝了后说：“火候过了，70分。”AI调整：下次缩短炒制时间。
•问题：每次只能得到一个模糊的分数，进步慢，而且如果第一次你心情不好，打分偏严，AI可能就懵了。

2. 用GRPO方法（小班课名师）：
•一次练习：你让AI厨师用同样食材，同时炒出8盘西红柿炒鸡蛋。它通过微调火候、调料顺序、翻炒手法等，做出了8盘略有差异的菜。
•组内评比：你把8盘菜摆在一起品尝、比较。
  。第3盘：色香味俱全！（组内排名第1）
  。第5盘：太甜了，奇怪。（组内垫底）
  。第1盘：还行，中等水平。（组内排名第4）
•精准反馈：你告诉AI：“第3盘的做法非常好，尤其是先炒蛋再盛出来的步骤，要保持！第5盘放糖的思路是错的，以后要避免。其他几盘的做法普普通通。”
•效果：AI通过这一次练习，就清晰地知道了“什么做法是好的”、“什么做法是差的”，以及“什么做法是一般的”。它学习的速度和效果远胜于一次只做一盘菜的传统方法。
