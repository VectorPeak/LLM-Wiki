## 1.自我介绍
自我介绍是面试的第一流程，也是你给面试官的第一印象。请不要讲流水账，把简历的内容照着说一遍。他并不是要关注你是谁，而是关注你和岗位的一个匹配度。时间请控制在 1 分钟 ~ 2 分钟，时间非常短，切记讲废话，直接灌输他想要的答案。别说什么籍贯、爱好，别说什么我很努力很认真空话虚话。在有限的时间内灌输重点

### 参考模板
1. 亮身份（学校/公司） + 核心标签：eg. 我叫 xx，我做过 2-3 年 AI 开发，我擅长 Agent、RAG 系统，熟悉常见的 AI 应用和框架，对模型前后训练、推理也有一定的实践
2. 用结果证明能力：我用 xx 方法得到了 xx 提升。使用我做的 RAG 系统，相比 copilot 通用工具，回答的时间降低 xx，准确度提升 xx 
3. 体现诚意：我了解过贵公司整个项目，我认为我的技术和它非常匹配，我也特别认可贵公司技术发展方向，特别期待加入这个岗位

~~~
Q:请做一段自我介绍

A:
(亮身份) 面试官您好, 我叫朱星岳, 来自西安电子科技大学. 过去主要从事 AI 应用开发，重点方向是 Agent、RAG 系统和企业知识库检索。最近我在 Windows 平台开发团队中，负责设计并落地了一套基于 Agent + RAG + MCP 的智能知识助手，用来解决团队版本发布信息分散、工程师排查效率低、客户问题响应慢等问题
 
在这个项目里，我参与了从架构设计到工程落地的完整过程。系统支持 PDF、Markdown 等多格式文档解析，结合 BM25、Dense Embedding 和 Cross-Encoder Reranker 做混合检索与精排；同时通过 MCP 协议把标准化工具暴露给 Agent 调用，实现跨文档语义检索、版本信息问答、补丁说明查询和兼容性问题排查。为了适配不同网络和模型环境，我也做了 LLM、Embedding、Reranker、VectorStore 的统一抽象，支持 Azure OpenAI、DeepSeek、Ollama 等模型一键切换，并引入 Redis 缓存来优化高频查询性能

(用结果证明能力) 结果上，这套系统支撑了 5000 多篇文档的语义检索[有点车]，Hit Rate@10 达到 92%，端到端查询延迟控制在 80ms 以内，文档查找时间缩短 50% 以上。同时我也搭建了 React Dashboard 管理平台，用于数据浏览、Ingestion 追踪、查询追踪和评估面板，并集成 Ragas 与 Golden Test Set 做回归测试，持续监控检索质量

(体现诚意) 我认为自己比较匹配贵公司这个岗位的原因是，我不仅熟悉常见 AI 应用框架，也真正做过 Agent、RAG、Embedding、Tool Calling、MCP、评估体系和工程化部署这一整条链路。相比只停留在 Demo 层面的 AI 应用，我更关注系统是否能稳定接入真实业务流程、是否可观测、可评估、可持续迭代。也正因为了解贵公司的技术方向，我很期待有机会把这些经验继续应用到更复杂、更有价值的产品场景中
~~~
