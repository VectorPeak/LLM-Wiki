一、基本概念定义

1、全精度训练（Full Precision Training）  
（1）通常指使用 **FP32** （32位浮点数）进行所有计算，包括前向传播、反向传播、参数更新；  
（2）FP32 提供约 7 位十进制有效数字，动态范围大，数值稳定性高；  
（3）是早期深度学习的标准训练方式。

2、混合精度训练（Mixed Precision Training, AMP）  
（1）在训练过程中 **同时使用 FP16** （16位浮点）；  
（2）核心思想：用 FP16 加速计算和节省显存，用 FP32 保证关键部分的数值稳定；  
（3）由 NVIDIA 在 Volta 架构后推广，现已成为大模型训练的标配。

二、技术实现与工作原理

1、数据类型分配策略

（1） **FP16 存储与计算** ：  
  ① 模型参数副本（主权重）、激活值、梯度通常以 FP16 存储；  
  ② 矩阵乘、卷积等算子在 Tensor Core 上以 FP16 执行，速度提升 2–3 倍；  
（2） **FP32 保留关键状态** ：  
  ③ 优化器维护的主权重（master weights）始终为 FP32；  
  ④ 梯度累加、参数更新在 FP32 中进行，避免舍入误差累积。

2、损失缩放（Loss Scaling）

（1）问题：FP16 动态范围小（最小正数约 6e-5），小梯度易下溢为零；  
（2）解决方案：将 loss 乘以一个 scale factor（如 2^16=65536），放大梯度；  
（3）反向传播后，再将梯度除以相同因子，恢复原始量级；  
（4）现代框架（如 PyTorch AMP）自动管理动态 loss scaling，无需手动调参。

3、硬件支持

（1）NVIDIA GPU 的 Tensor Core 对 FP16 矩阵运算有专用硬件加速；  
（2）Ampere 架构（A100）及以后还支持 BF16（Brain Float16），其指数位与 FP32 相同，更不易溢出；  
（3）无 Tensor Core 的设备（如 CPU、旧 GPU）无法从混合精度获益。

三、性能与资源对比

1、显存占用

（1）FP16 参数、梯度、激活值大小约为 FP32 的一半；  
（2）以 7B 模型为例：  
  ① 全精度（FP32）仅参数就占 28 GB；  
  ② 混合精度（FP16 主权重 + FP32 优化器）可将模型相关显存降低 30%–50%；  
（3）显存节省使得更大 batch size 或更长序列成为可能。

2、训练速度

（1）FP16 计算吞吐通常是 FP32 的 2–3 倍（依赖硬件）；  
（2）实际端到端加速比约为 1.5–2.5 倍，受限于数据加载、通信等非计算瓶颈；  
（3）在千卡集群中，混合精度对整体训练周期缩短效果显著。

3、数值稳定性与收敛性

（1）合理配置下，混合精度模型收敛曲线与全精度几乎一致；  
（2）若 loss scaling 不当或模型本身梯度极小（如某些 GAN），可能出现 NaN 或发散；  
（3）大语言模型因 loss 较大、梯度分布较广，通常对混合精度非常友好。

四、实际应用中的选择建议

1、何时使用混合精度  
（1）训练大模型（>1B 参数）时几乎必须启用；  
（2）GPU 支持 Tensor Core（如 V100、A100、H100）；  
（3）任务对微小数值误差不敏感（如语言建模、分类）。

2、何时谨慎使用或禁用  
（1）科研实验需严格复现 FP32 结果；  
（2）模型包含极端小梯度操作（如某些物理仿真）；  
（3）使用不支持 AMP 的自定义 CUDA kernel。

3、框架支持  
（1）PyTorch： `torch.cuda.amp.autocast()` + `GradScaler` ；  
（2）DeepSpeed：内置 AMP，一键开启；  
（3）Hugging Face Transformers：通过 `fp16=True` 或 `bf16=True` 配置。
