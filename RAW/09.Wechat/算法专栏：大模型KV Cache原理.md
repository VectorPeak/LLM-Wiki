KV Cache（键值缓存）是大模型推理加速中的一项关键技术。其核心思想是通过缓存历史计算结果来避免重复计算，即用额外的存储空间换取时间效率。具体而言，在自回归生成过程中，模型每生成一个新词元时，无需重新计算之前所有词元的键值矩阵，而是直接从缓存中读取已计算的结果。

Transformer架构采用自回归方式进行生成，即生成第 N 个词元时需要依赖前 N-1 个词元的信息。如果不使用 KV Cache，模型每生成一个新词元，都必须从头开始重新计算从第一个到当前词元的所有输入，导致计算量随序列长度呈平方级增长。引入 KV Cache 后，模型将预先计算好的 Key 与 Value 矩阵保存在显存中，生成新词元时仅需计算当前词元的 Query 向量，然后将其与缓存中的历史 K 和 V 矩阵进行注意力运算，从而使计算量降至线性复杂度。

在 Transformer 的注意力机制中，计算公式为

Attention(Q, K, V) = softmax(Q K^T / sqrt(d\_k)) V

推理过程分为两个阶段。预填充阶段处理输入提示词时，模型一次性计算出所有词元的 K 与 V 矩阵并存入 KV Cache。解码阶段生成第 t 个词元时，模型仅计算当前词元的 Q\_t、K\_t、V\_t，将新的 K\_t 与 V\_t 追加到缓存中，再以当前 Q\_t 与缓存中所有历史的 K 矩阵进行点积运算，获取上下文权重后与 V 矩阵加权求和。

KV Cache 虽然提高了推理速度，但会消耗大量显存。其显存占用量取决于模型的层数、注意力头数、头维度、序列长度以及数据类型。显存占用的计算公式为

Memory = 2 × Layers × SeqLen × Heads × HeadDim × DtypeSize

其中系数 2 代表 Key 与 Value 两个矩阵，DtypeSize 表示数据类型的字节数（FP16 为 2 字节，FP32 为 4 字节）。以 LLaMA-7B 模型为例，该模型包含 32 层，32 个注意力头，头维度为 128，采用 FP16 精度处理序列长度 2048，单层占用约 21 MB，总显存占用约为 671 MB。实际工程实现中，因内存对齐等因素，占用可能更高。随着序列长度增加，KV Cache 的大小会线性增长，在某些场景下可能超过模型权重本身的显存占用。

针对 KV Cache 显存占用过大的问题，当前业界提出了若干优化方案。分组查询注意力（GQA）允许多个 Query 头共享同一组 Key 与 Value 头，从而减少缓存数据量，已在 LLaMA-3、Mistral 等模型中得到应用。PagedAttention 技术借鉴操作系统的虚拟内存分页思想，通过解决显存碎片化问题支持高效的连续批处理，代表性框架为 vLLM。KV Cache 量化将 FP16 精度的缓存数据压缩为 INT8 或 FP8 格式，在精度损失极小的情况下将显存占用减半，TensorRT-LLM 等框架支持该技术。滑动窗口方法仅保留最近若干个词元的 KV，丢弃久远的历史信息，适用于对话场景，Mistral 与 Gemma 等模型采用此策略。多级缓存与卸载技术将不常用的 KV Cache 从 GPU 显存交换至 CPU 内存或 NVMe 固态硬盘中，例如 DeepSeek 的 HCA/CSA 架构。
