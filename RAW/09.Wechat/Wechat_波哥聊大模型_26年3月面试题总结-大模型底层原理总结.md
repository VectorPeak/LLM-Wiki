## 一、Self-Attention：面试的绝对中心

Self-Attention 相关问题占据了整份题库的近 40%。面试官从计算流程、缩放因子、复杂度分析、Q/K/V 设计动机四个角度反复考察。

### 1.1 Self-Attention 的完整计算流程

**考察意图：** 入门必答，但面试官期望你能精确到张量维度和每一步的设计动机。

**回答框架：**

给定输入 X ∈ ℝ^{n×d\_model}（n 为序列长度，d\_model 为隐藏维度），Self-Attention 的计算流程为：

**第一步：线性投影生成 Q/K/V。**

其中 W^Q, W^K ∈ ℝ^{d\_model × d\_k}，W^V ∈ ℝ^{d\_model × d\_v}。三个独立投影矩阵将输入映射到不同的语义子空间——Q 表示"我在找什么"，K 表示"我能提供什么"，V 表示"我实际携带的信息"。

**第二步：计算缩放点积注意力分数。**

QK^T ∈ ℝ^{n×n} 产生 token 间的相似度矩阵，除以 √d\_k 防止点积值过大导致 softmax 饱和（详见 1.2），softmax 将每行归一化为概率分布，最终与 V 加权求和得到输出。

**第三步（如有因果掩码）：** 在 softmax 前对注意力分数矩阵的上三角区域加负无穷，确保位置 i 只能关注位置 ≤ i 的 token。

输出维度为 ℝ^{n×d\_v}，与输入序列长度相同，但每个位置的表征已融合了全序列的上下文信息。

### 1.2 为什么 QK^T 要除以 √dₖ？

**考察意图：** 出现 **15+ 次** ，是整份题库中出现频率最高的单一问题。面试官通过追问深度区分候选人水平。

**回答框架（三层递进）：**

**第一层（概率分布视角）：** 假设 Q 和 K 的各分量独立同分布，均值为 0、方差为 1，则 q·k = Σᵢ qᵢkᵢ 的方差为 dₖ。当 dₖ 较大时（如 64 或 128），点积值的绝对值会很大，导致 softmax 的输入分布变得极端——几乎所有概率集中在最大值对应的位置，形成近似 one-hot 分布。这使得注意力失去"软选择"的能力，模型退化为硬性地只关注一个 token。除以 √dₖ 将方差归一化回 1，保持 softmax 输出的"温和"分布。

**第二层（梯度稳定性视角）：** softmax 在输入值极大或极小时梯度趋近于零（饱和区）。未缩放的大点积值会将 softmax 推入饱和区，导致反向传播时梯度消失，注意力权重几乎无法被更新。缩放因子确保 softmax 的输入保持在梯度敏感区间内。

**第三层（初始化兼容性视角）：** 标准的 Xavier/He 初始化假设各层输出的方差为 O(1)。如果不缩放，注意力分数的方差随 dₖ 增长，与初始化的方差假设不匹配，破坏了整个网络的数值稳定性链条。

**加分追问：** 若将 1/√dₖ 替换为可学习标量，理论上可以让模型自适应调整缩放强度，但实际中引入了训练初期不稳定的风险——可学习标量可能在早期被更新到极端值，且不同层/头需要不同的缩放，增加了优化难度。

### 1.3 Self-Attention 的 O(n²) 复杂度来源

**考察意图：** 高频题，考察对计算瓶颈的精确定位。

**回答框架：**

O(n²) 主要来自两个操作：

**QK^T 矩阵乘法：** Q ∈ ℝ^{n×d\_k} 与 K^T ∈ ℝ^{d\_k×n} 相乘，产生 n×n 的注意力分数矩阵。计算量为 O(n² · d\_k)，空间开销为 O(n²)。

**注意力加权求和：** softmax(QK^T/√d\_k) ∈ ℝ^{n×n} 与 V ∈ ℝ^{n×d\_v} 相乘，计算量为 O(n² · d\_v)。

由于 d\_k 和 d\_v 通常为常数（如 64），主导复杂度为 O(n²)。当序列长度 n 增长到万级别以上时，这个平方复杂度成为训练和推理的核心瓶颈。

**主流缓解方法：**

- **FlashAttention：** 不降低理论复杂度，而是通过 tiling 和 kernel fusion 减少 HBM 访问次数，将实际吞吐提升 2-4 倍。它将 O(n²) 的显存降低为 O(n)。
- **稀疏注意力（如 Sliding Window、Sparse Transformer）：** 限制每个 token 只关注局部窗口或固定模式的 token 子集，将复杂度降至 O(n · w)，w 为窗口大小。
- **线性注意力（如 Linformer、Performer）：** 用核函数近似 softmax，将 QK^T 的计算改写为 φ(Q) · (φ(K)^T · V)，利用矩阵乘法结合律将复杂度降至 O(n · d)。代价是近似误差可能影响建模精度。

### 1.4 为什么 Q/K/V 需要三个独立的线性投影？

**考察意图：** 高频题（8+ 次），考察对注意力机制设计哲学的理解。

**回答框架：**

三个独立投影的设计动机是 **功能解耦** ：

- **Q（Query）：** 编码"当前位置想要查询什么信息"——是一种需求表达。
- **K（Key）：** 编码"当前位置能提供什么信息"——是一种供给描述。
- **V（Value）：** 编码"当前位置实际携带的内容"——是实际被聚合的信息。

如果共享 Q 和 K（即使用 QQ^T V），注意力矩阵 QQ^T 变为对称半正定矩阵，这意味着：token A 对 token B 的关注度必然等于 B 对 A 的关注度。这个对称性约束严重限制了模型的表达能力——自然语言中的依赖关系通常是非对称的（例如代词依赖其先行词，但先行词不一定依赖代词）。

如果共享所有三个投影（Q=K=V），则注意力退化为输入自身的加权平均，失去了"用一种视角查询、用另一种视角回答"的灵活性。

### 1.5 多头注意力的设计动机与优势

**考察意图：** 出现 10+ 次，考察对"多头"设计的深层理解。

**回答框架：**

多头注意力的核心思想是：将 d\_model 维度拆分为 h 个头，每个头在 d\_k = d\_model/h 维的子空间中独立计算注意力，最后拼接并投影：

相比单头注意力的三个本质优势：

**子空间解耦：** 不同头可以关注不同类型的关系——一个头可能捕捉语法依赖（如主谓关系），另一个头捕捉语义相似性，第三个头捕捉位置邻近性。单头注意力将所有关系压缩到同一个注意力分布中，不同类型的依赖会相互干扰。

**表征多样性：** 多个头并行地在不同子空间中建模，增加了整体表征的丰富度。即使某些头退化（如过度关注局部模式），其他头仍能提供互补信息。

**并行计算友好：** 多个头的计算是完全独立的，可以在 GPU 上高效并行执行。总计算量与单头相同（因为 d\_k = d\_model/h），但表达能力显著更强。

拼接后的线性投影 W^O 的作用是：将各头在不同子空间中学到的表征重新映射回 d\_model 维空间，实现跨头的信息融合。

## 二、位置编码：从正弦到 RoPE

位置编码是仅次于 Self-Attention 的第二大考点。面试题从最基础的"为什么需要位置编码"到 RoPE 的数学推导，覆盖完整的知识梯度。

### 2.1 为什么 Transformer 需要位置编码？

**考察意图：** 基础但重要，好的回答要点明 Self-Attention 的置换等变性。

**回答框架：**

Self-Attention 的计算过程是 **置换等变的（permutation equivariant）** ——如果打乱输入序列的 token 顺序，输出也会以同样方式打乱，但每个位置的输出值不变。换言之，Self-Attention 本身无法区分 "猫 吃 鱼" 和 "鱼 吃 猫"。

位置编码通过向输入中注入位置信息，打破这种置换等变性，使模型能感知 token 的顺序。

**相对位置可线性表达：** 对于固定偏移 k，PE(pos+k) 可以表示为 PE(pos) 的线性变换（旋转矩阵），这使模型有可能学会捕捉相对位置关系。

### 2.3 RoPE 的核心原理

**考察意图：** 2026 年最高频的位置编码考点（10+ 次），常要求写出数学形式甚至手写实现。

**为什么 RoPE 外推性优于绝对位置编码？** 旋转变换是连续、周期性的，位置 m 的编码是对基频率的线性调制，理论上可以平滑地延伸到任意位置。而可学习的绝对位置编码是一个查找表，超出训练长度时没有对应的参数。

**RoPE 如何保持语义不变性？** 旋转变换保持向量的模长不变（正交变换），因此不会改变 token 嵌入的"语义强度"，只改变其在子空间中的方向，用方向编码位置。

## 三、归一化方法：BatchNorm → LayerNorm → RMSNorm

归一化是"小而重要"的考点。核心问题就两个：为什么 NLP 不用 BatchNorm？为什么 LLaMA 用 RMSNorm 而非 LayerNorm？

### 3.1 BatchNorm 为何不适合 Transformer

**考察意图：** 出现 6+ 次，考察对归一化维度与数据特性匹配关系的理解。

**回答框架：**

BatchNorm 沿 batch 维度计算均值和方差，这在 NLP 的 Transformer 中存在三个根本性问题：

**变长序列导致统计量不稳定。** 同一 batch 中不同样本的序列长度不同，位置 i 处的 BatchNorm 统计量来自不同样本在该位置的 token——有的样本此处是实际内容，有的已经是 padding。这使得归一化基于的是一个混杂了真实信号和无意义 padding 的统计量。

**小 batch size 下方差估计不可靠。** 大模型训练中由于显存限制，batch size 通常很小（如 1-4）。BatchNorm 在小 batch 下的统计量方差极大，导致训练不稳定。

**自回归推理时 batch 语义丢失。** 推理时通常 batch size=1，且逐 token 生成，BatchNorm 的运行均值/方差（来自训练阶段）与推理时的分布可能不匹配。

LayerNorm 沿特征维度（d\_model）计算统计量，完全独立于 batch 和序列其他位置，天然适配变长序列和小 batch 场景。

**计算效率：** 省去了均值计算和减法操作，减少约 20-30% 的归一化计算量。在大模型中，归一化操作虽然单次开销小，但每层的每个子模块都要执行，累积效果显著。

**训练稳定性：** 实证表明，均值中心化对 Transformer 的训练效果贡献有限——深层 Transformer 的隐藏状态分布通常已经接近零均值，去除均值中心化不会显著影响模型质量。

**参数简化：** RMSNorm 去除了偏置参数 β，进一步减少参数量。

LLaMA 系列选择 RMSNorm 正是基于这个"同等效果下更简更快"的工程判断。

### 3.3 Pre-Norm vs Post-Norm

**考察意图：** 出现 7+ 次，考察对归一化位置选择的架构理解。

**回答框架：**

两者的区别在于 LayerNorm（或 RMSNorm）相对于残差连接的位置：

**Post-Norm（原始 Transformer）：** `x + LN(SubLayer(x))` 归一化放在残差连接之后。

**Pre-Norm（LLaMA 等主流大模型）：** `x + SubLayer(LN(x))` 归一化放在子层（Attention/FFN）输入之前。

Pre-Norm 的核心优势在于梯度流动：

在 Pre-Norm 中，残差路径 `x → x + SubLayer(LN(x))` 是一条"干净"的恒等映射——梯度可以通过残差连接直接从深层流回浅层，不经过归一化层的缩放。这使得极深的网络（如 80+ 层）仍能保持稳定的梯度幅值。

在 Post-Norm 中，梯度必须经过归一化层，归一化层的 Jacobian 矩阵会改变梯度的幅值和方向。在深层网络中，这种效应累积会导致底层梯度要么过小（难以更新）要么方差过大（不稳定）。

代价是：Pre-Norm 的最终性能天花板在某些任务上略低于 Post-Norm（因为归一化在子层输入前会"压缩"输入的尺度，可能损失表达能力）。但对于大模型训练而言，稳定性远比微小的性能差距重要。

## 四、FFN、激活函数与架构组件

这类题考察对 Transformer 中 Self-Attention 以外组件的理解，区分"只知道 Attention"和"理解完整架构"的候选人。

### 4.1 FFN 层的核心作用

**考察意图：** 出现 5+ 次，考察对 FFN 不可替代性的理解。

**回答框架：**

FFN 在 Transformer 中承担的核心功能是 **逐位置的非线性特征变换** ：

其中中间维度通常是 d\_model 的 4 倍（升维-激活-降维）。

如果移除 FFN 只保留多头注意力，模型会丧失几个关键能力：

**非线性变换能力缺失。** Self-Attention 本质上是输入的加权线性组合（softmax 提供的非线性只作用于权重，不作用于值）。FFN 中的激活函数（如 SwiGLU）提供了真正的逐元素非线性变换，是模型逼近复杂函数的关键。根据 Universal Approximation 定理，足够宽的两层 FFN 可以逼近任意连续函数。

**特征维度适配。** FFN 的升维操作（d → 4d → d）让模型在高维空间中进行特征变换，有效扩展了每个位置的表征容量。

**位置无关的模式提取。** Self-Attention 建模 token 间的交互，FFN 则对每个位置独立地提取特征模式。两者互补——Attention 负责"看哪里"，FFN 负责"怎么处理看到的内容"。

**门控机制：** SwiGLU 引入了乘性门控——一个分支（xW₁ 经过 Swish）作为"门"，控制另一个分支（xW₂）的信息流通量。这种门控机制比单纯的逐元素激活提供了更精细的特征选择能力。

**梯度特性：** Swish 是平滑的非单调函数，在负值区域有小的负输出（而非 ReLU 的硬截断），避免了"死神经元"问题。门控乘法进一步使梯度路径更丰富。

**实证效果：** 在相同参数量下，SwiGLU 相比 GeLU 通常带来 0.5-1% 的下游任务提升。代价是需要两个投影矩阵（W₁ 和 W₂），参数量略增，但总体参数效率更高（因为效果提升的边际大于参数增量）。

### 4.3 残差连接的作用与深层网络的挑战

**考察意图：** 基础但重要，好的回答要超越"缓解梯度消失"这句话。

**回答框架：**

残差连接 `y = x + F(x)` 在深层 Transformer 中解决三个层级的问题：

**梯度传播层面：** 创建了一条"快捷路径"，梯度可以跳过中间的非线性变换层直接回传。即使 F(x) 的梯度接近零，梯度仍可通过恒等映射路径流动。在 L 层网络中，无残差连接时梯度需经过 L 次矩阵乘法（可能指数衰减），有残差连接时每层梯度至少包含一个 O(1) 的分量。

**特征复用层面：** 残差连接确保浅层的特征在深层中始终可访问，防止"信息瓶颈"——深层不必从头重建浅层已经提取的模式。

**优化 landscape 层面：** 残差连接使损失函数的 landscape 更平滑（Hessian 特征值分布更均匀），降低了优化难度。

**超深层的固有局限：** 标准残差连接在超深网络（如 100+ 层）中仍存在问题。信息稀释——每层只贡献 F(x)，而直通路径 x 持续累积，深层的贡献比例越来越小。缓解方案包括 LayerScale（对 F(x) 施加可学习的缩放因子，初始化为小值）和 Stochastic Depth（训练时随机跳过某些层，增强正则化）。

## 五、架构选型：为什么是 Decoder-only？

"为什么主流大模型用 Decoder-only" 是出现频率第三高的问题（8+ 次），且答案的深度直接反映候选人对架构设计的理解层次。

### 5.1 Decoder-only vs Encoder-only vs Encoder-Decoder

**考察意图：** 必考综合题，需要从多个维度展开对比。

**回答框架：**

| 维度 | Encoder-only (BERT) | Decoder-only (GPT/LLaMA) | Encoder-Decoder (T5) |
| --- | --- | --- | --- |
| 注意力掩码 | 全双向（每个 token 看到所有其他 token） | 因果掩码（只看到左侧 token） | Encoder 双向 + Decoder 因果 + 交叉注意力 |
| 训练目标 | MLM（预测被 mask 的 token） | 自回归 NTP（预测下一个 token） | Seq2Seq（编码输入 → 生成输出） |
| 天然适配任务 | 判别式（分类、NER、相似度） | 生成式（文本生成、对话、推理） | 条件生成（翻译、摘要） |
| 训练效率 | MLM 只在 ~15% mask token 上计算 loss | 每个 token 都贡献 loss | Decoder 部分每个 token 贡献 loss |

### 5.2 Decoder-only 成为主流的核心原因

**回答框架：**

**训练目标与生成任务天然一致。** 自回归 Next Token Prediction 与推理时逐 token 生成的过程完全相同，不存在训练-推理不一致的问题。BERT 的 MLM 训练时有 \[MASK\] token，推理时没有，存在天然的 mismatch。

**每个 token 都是训练信号。** 自回归目标对序列中的每个位置都计算 loss，数据利用效率远高于 MLM（只对 15% 的 mask 位置计算 loss）。在大规模预训练中，这意味着同样的数据量能提供更多的训练信号。

**统一的推理接口。** 所有任务（分类、生成、推理、工具调用）都可以统一为"给定前缀，生成续写"的范式。Encoder-only 需要为不同任务设计不同的 head，Encoder-Decoder 需要区分输入和输出，Decoder-only 的接口最简洁。

**扩展性与工程效率。** 单一的 Decoder 堆叠结构在分布式训练中更容易做流水线并行和张量并行。KV Cache 的复用机制使自回归推理效率可控。

### 5.3 因果掩码如何实现自回归约束

**考察意图：** 偏实现层面，常与代码题结合。

**回答框架：**

因果掩码通过在 softmax 前对注意力分数矩阵的上三角区域加 -∞ 来实现。具体地：

加 -∞ 后经过 softmax，这些位置的注意力权重变为 0（因为 e^{-∞} = 0），确保位置 i 的输出只依赖于位置 1 到 i 的输入。

为什么在 softmax **前** 加掩码而非 **后** 置零？如果在 softmax 后置零，剩余位置的权重之和不为 1，违反了概率分布的归一化约束，且梯度传播路径会包含被置零位置的影响。在 softmax 前用 -∞ 掩码是数学上干净的方案——softmax 自然地将这些位置的概率归零且保持剩余位置的归一化。

## 六、MoE 架构

MoE（Mixture of Experts）是 2026 年的新兴高频考点，随着 Mixtral、DeepSeek-V2/V3 等模型的广泛部署而成为面试热门。

### 6.1 MoE 的核心思想与权衡

**考察意图：** 基础必答题。

**回答框架：**

MoE 的核心思想是 **计算量与模型容量解耦** 。标准 dense 模型中，每个 token 都经过所有参数的计算；MoE 模型中，每个 token 只激活部分"专家"（通常是 FFN 的变体），其他专家不参与计算。

这意味着：模型总参数量可以远大于单个 token 实际使用的参数量。例如 Mixtral 8×7B 有约 47B 总参数，但每个 token 只激活 2 个专家，实际计算量约等于 14B 模型。

核心权衡是：更大的模型容量（更多参数存储更多知识） vs 路由和通信的额外开销。

### 6.2 路由机制与负载均衡

**考察意图：** 出现 6+ 次，考察对 MoE 训练难点的理解。

**回答框架：**

Top-k 路由的计算过程：

对输入 token x 通过一个可学习的门控网络 Wg 计算每个专家的选择概率，只将 token 发送到概率最高的 k 个专家。

**路由坍塌问题：** 训练过程中，少数专家可能获得大部分 token（"赢者通吃"），导致其他专家得不到训练信号逐渐退化。这是一个自强化的正反馈循环——越常被选中的专家越优秀，越优秀越常被选中。

**缓解策略：**

- **辅助损失（Auxiliary Load Balancing Loss）：** 在主损失之上增加一个鼓励均匀分配的正则项，惩罚专家负载的不均衡度。
- **Expert Choice 路由：** 反转路由方向——不是让 token 选择专家，而是让每个专家选择固定数量的 token。这天然保证了负载均衡。
- **随机路由噪声：** 在门控分数上添加噪声（如 Noisy Top-k），增加路由的探索性，防止过早收敛到固定的路由模式。

### 6.3 MQA/GQA/MHA/MLA 的对比

**考察意图：** 出现 8+ 次，考察对注意力头结构演进的理解。

**回答框架：**

四者的核心区别在于 K/V 头的共享程度：

**MHA（Multi-Head Attention）：** 每个 Q 头有独立的 K/V 头。KV Cache 大小 = h × d\_k × 2 × seq\_len。表达能力最强，显存最大。

**MQA（Multi-Query Attention）：** 所有 Q 头共享一组 K/V。KV Cache 缩减为 1/h。推理速度最快，但表达能力损失明显。

**GQA（Grouped-Query Attention）：** Q 头分为 g 组，每组共享一组 K/V。KV Cache 缩减为 g/h。是 MHA 和 MQA 的折中——LLaMA 2 70B 采用 GQA (g=8) 在精度和效率间取得了良好平衡。

**MLA（Multi-head Latent Attention，DeepSeek-V2）：** 将 K/V 投影到低维潜在空间（latent space），通过低秩压缩大幅减少 KV Cache 的同时保持表达能力。理论上比 GQA 更灵活，但实现复杂度更高。

选择依据：训练阶段不敏感（都可训练）→ 区别主要在推理。推理显存和吞吐为首要约束 → MQA 或 GQA；追求精度且显存允许 → MHA；需要极致的 Cache 压缩 → MLA。

## 七、进阶专题

### 7.1 Temperature 对生成行为的影响

**考察意图：** 出现 4+ 次，考察对采样机制的数学理解。

**回答框架：**

Temperature τ 作用于 softmax 前的 logits：

τ → 0 时：分布趋近于 one-hot（只选概率最大的 token），生成高度确定但缺乏多样性，重复风险高。

τ → ∞ 时：分布趋近于均匀分布，生成高度随机，质量不可控。

τ = 1 时：使用模型原始的 logits 分布。

本质上，temperature 控制的是输出分布的 **熵** ——τ 小则低熵（确定性），τ 大则高熵（多样性）。实践中对话场景通常用 τ ∈ \[0.7, 1.0\]，创意写作用 τ ∈ \[1.0, 1.5\]，代码生成用 τ ∈ \[0.2, 0.5\]。

### 7.2 Transformer 参数量估算

**考察意图：** 笔试高频题，面试官期望你能推导出具体数字。

**回答框架：**

以标准 Decoder-only 架构（如 LLaMA）为例，每层参数包含：

**Self-Attention：** Q/K/V/O 四个投影矩阵，各为 d × d，总计 4d²。

**FFN：** 标准的两层 MLP 为 d × 4d + 4d × d = 8d²。若使用 SwiGLU，则需要三个矩阵（门控分支 + 值分支 + 下投影）：3 × d × (8d/3) ≈ 8d²（SwiGLU 通常调整中间维度以保持总参数量相当）。

**归一化层：** RMSNorm 每层有 2 × d 个参数（两个归一化层各 d 个），可忽略不计。

**每层总计：** 约 12d²（标准 FFN）。

**全模型：** L 层 × 12d² + 词表嵌入层（V × d）+ 最终 LM Head（d × V，通常与嵌入层共享权重）。

以 LLaMA-7B 为例：d=4096，L=32，V=32000。 每层参数 ≈ 12 × 4096² ≈ 201M，32 层 ≈ 6.4B，嵌入层 ≈ 32000 × 4096 ≈ 131M，总计约 6.7B（加上其他小组件约 7B）。

### 7.3 CoT 推理为何有效？

**考察意图：** 考察对 CoT 作用机制的深层理解。

**回答框架：**

Chain-of-Thought 之所以能提升复杂推理任务的表现，核心原因不是"教会了模型推理"，而是 **通过延长生成序列扩展了模型的有效计算量** 。

标准自回归生成中，模型对下一个 token 的预测只能利用 Transformer 固定层数的前向传播——这限制了单步推理的"深度"。CoT 通过要求模型先输出中间推理步骤，实质上将一个需要多步推理的问题分解为多个单步预测问题，每一步都可以利用前面步骤的输出作为"工作记忆"。

从信息论的角度：CoT 将隐式推理（在模型内部的隐藏状态中完成）转化为显式推理（在 token 序列中展开），使中间结果可被后续步骤直接"看到"（通过 Attention），而非被压缩在有限维度的隐藏状态中。
