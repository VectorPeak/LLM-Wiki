## 0x01. 大模型算法：ViT结构
> 发布日期：2026-05-27  
> 原文链接：[大模型算法：ViT结构](https://mp.weixin.qq.com/s/DeuZaV3N6Q7PBDORqbW6vQ)
### 1. ViT的基本思想与背景
Vision Transformer(ViT)是2020年由Google Research提出的一种将纯Transformer架构应用于图像识别任务的模型。其核心思想是：将图像视为一个“token序列”，从而直接复用在自然语言处理中取得巨大成功的Transformer编码器。

传统卷积神经网络(CNN)依赖局部感受野和权重共享来提取空间特征，而ViT则完全摒弃卷积操作，仅通过自注意力机制建模全局像素间关系。这一设计打破了计算机视觉对归纳偏置(如平移不变性、局部性)的依赖，证明了大规模数据下纯注意力模型的有效性。

### 2. ViT的整体架构流程
#### 1. 图像分块(Patch Embedding)
(1)输入图像被划分为固定大小的非重叠块(patches)，例如 16×16 像素；(2)每个 patch 被展平为一维向量，维度为(P 为 patch 边长)；(3)通过一个可学习的线性投影矩阵，将每个 patch 映射到 D 维嵌入空间，得到 patch embeddings；(4)最终得到 N 个 token，其中。

### 2. 添加位置编码(Positional Encoding)
(1)由于 Transformer 本身不具备顺序感知能力，需显式注入位置信息；(2)ViT 采用可学习的一维位置编码(而非正弦函数)，与 patch embeddings 相加；(3)位置编码数量为 N + 1，额外的一个用于 class token。

### 3. Class Token 机制
(1)在序列开头插入一个特殊的可学习向量

token；(2)该 token 在 Transformer 各层中与其他 patch tokens 一起参与自注意力交互；(3)最终，取

token 对应的输出作为整图的全局表示，用于分类头预测。

### 4. Transformer 编码器堆叠
(1)由 L 层标准 Transformer 编码器组成，每层包含：1. 多头自注意力模块(MSA)；2. 层归一化(LayerNorm)；3. 前馈网络(MLP，通常为两层全连接 + GELU 激活)；(2)残差连接应用于 MSA 和 MLP 子层；(3)所有层共享相同的 hidden dimension D。

### 5. 分类头(Classification Head)
(1)取最后一层

token 的输出；(2)通过一个线性分类器(通常带 dropout)映射到类别 logits；(3)在预训练阶段使用大 softmax 分类，微调时可替换为任务特定头。

#### 3. ViT的关键特性与影响
#### 1. 对数据规模的高度依赖
(1)ViT 在 ImageNet-1k 上表现不如 CNN，但在 JFT-300M 等超大规模数据集上显著超越；(2)表明纯注意力模型需要大量数据才能学习有效的视觉先验；(3)这也解释了为何 ViT 成为多模态大模型(如 CLIP、BLIP)的首选视觉编码器——它们天然拥有海量图文对数据。

#### 2. 全局建模能力
(1)自注意力机制允许任意两个 patch 直接交互，捕获长距离依赖；(2)相比 CNN 的局部感受野逐层扩大，ViT 从第一层就具备全局视野；(3)在需要整体结构理解的任务(如场景分类、姿态估计)中具有优势。

#### 4. 与 CNN 的对比
(1)CNN 具有强归纳偏置，小数据下泛化好，但表达能力受限；(2)ViT 归纳偏置弱，大数据下容量高，但对数据和算力要求高；(3)实践中，两者常结合使用(如 ConvNeXt、CoaT)，或在 ViT 中引入局部性(如 LocalViT)。

#### 4. ViT的变体与演进
#### 1. DeiT(Data-efficient Image Transformer)
(1)引入蒸馏 token，利用 CNN 教师模型指导训练；(2)在 ImageNet-1k 上实现高效训练，无需超大数据集。

#### 2. Swin Transformer
(1)采用滑动窗口机制，将自注意力限制在局部窗口内；(2)引入窗口间 shift 操作，实现跨窗口连接；(3)构建层次化特征图，适配密集预测任务(如检测、分割)。

#### 3. MAE(Masked Autoencoders)
(1)将 ViT 用于自监督预训练，随机掩码大量 patch 并重建；(2)证明 ViT 在无监督条件下也能学习高质量表征；(3)成为当前主流视觉预训练范式之一。

ViT 结构通过将图像转化为 token 序列，并利用 Transformer 编码器进行全局建模，开创了视觉领域“去卷积化”的新范式。尽管其原始版本对数据和算力要求较高，但其简洁性、可扩展性和与大语言模型的天然兼容性，使其成为多模态大模型和视觉基础模型的核心组件。

## 0x02. 大模型算法：不同形状图片的位置编码
> 发布日期：2026-05-27  
> 原文链接：[大模型算法：不同形状图片的位置编码](https://mp.weixin.qq.com/s/B8fIbEbML0uHZv4jykx4gw)
### 1. 问题背景与挑战
在基于 Vision Transformer(ViT)的多模态大模型中，图像通常被划分为固定大小的 patch(如 14×14 或 16×16 像素)，每个 patch 被视为一个 token。然而，真实场景中的图像具有任意宽高比和分辨率(如手机竖屏照片、网页横幅、文档扫描件)，导致 patch 序列长度和空间布局高度可变。

标准 ViT 使用固定数量的一维可学习位置编码，仅适用于固定分辨率输入(如 224×224 → 14×14=196 patches)。当处理不同形状图像时，直接使用预训练的位置编码会导致：
因此，如何设计适应任意图像形状的位置编码，成为提升多模态模型鲁棒性的关键问题。

### 2. 主流位置编码设计方案
#### 1. 二维可学习位置编码 + 插值
(1)将位置编码组织为二维网格形式(H_patch × W_patch)，而非一维序列；(2)对于新尺寸图像(H' × W')，将其位置编码通过双线性插值从原始网格(如 14×14)映射到目标网格；(3)该方法被 CLIP-ViT、DINO 等广泛采用，简单有效；(4)局限：插值在大幅外推时(如 224 → 1024)性能下降，因高频位置分量混叠。

### 2. 相对位置编码(Relative Position Encoding)
(1)不使用绝对位置，而是建模 token 之间的相对偏移(Δrow, Δcol)；(2)在自注意力计算中，将相对位置 bias 加入 attention logits；(3)代表工作：Swin Transformer、CoaT；(4)优势：天然支持任意尺寸，因相对关系与绝对大小无关；(5)挑战：需设计高效的相对位置索引机制，避免 O(N²) 存储开销。

### 3. RoPE(Rotary Position Embedding)的视觉扩展
(1)RoPE 原用于 LLM，通过旋转矩阵编码 token 的绝对位置，并隐含相对位置不变性；(2)在视觉领域，将图像 patch 的 (row, col) 坐标分别映射到两个正交旋转子空间；(3)例如，ViT-RoPE 或 Phi-3 Vision 中，每个 patch 的位置由其二维坐标生成旋转矩阵；(4)优点：具备良好的外推能力，且无需插值；(5)实现时需将 2D 坐标分解为独立维度处理，保持旋转性质。

### 4. 动态位置编码(Dynamic or Content-Aware)
(1)位置编码不再固定，而是由轻量子网络根据图像内容或形状动态生成；(2)例如，通过一个小 CNN 或 MLP 接收图像尺寸作为输入，输出适配的位置向量；(3)或结合注意力机制，让模型自适应地调整位置权重；(4)此类方法灵活性高，但增加训练复杂度，且可能降低稳定性。

### 5. 无位置编码(Position-Free)探索
(1)部分研究发现，在大规模预训练下，ViT 可部分从数据中学习位置先验；(2)但完全移除位置编码通常导致性能显著下降，尤其在细粒度任务上；(3)因此，实践中仍需显式位置信号，但可弱化其形式(如仅加在早期层)。

#### 3. 工程实践中的组合策略
#### 1. 训练阶段统一预处理
(1)将所有图像 resize 到固定短边(如 224)，再 center crop 或 pad；(2)虽损失部分信息，但保证位置编码一致性；(3)适用于对精度要求不高或部署受限的场景。

#### 2. 推理时自适应插值 + 边界填充
(1)对任意输入，先计算所需 patch 网格(H_p, W_p)；(2)若 H_p 或 W_p 超出预训练最大值，则进行插值扩展；(3)同时在位置编码边缘添加可学习 padding token 的位置向量，缓解边界效应。

#### 3. 多尺度位置编码池
(1)预训练多个分辨率下的位置编码(如 224、384、512)；(2)推理时根据输入尺寸选择最接近的编码，或进行加权融合；(3)增加存储开销，但提升跨尺度泛化能力。

面对不同形状的图像，位置编码设计需在泛化性、计算效率与实现复杂度之间权衡。当前主流方案以二维可学习编码 + 插值为基础，在多模态大模型(如 LLaVA、Qwen-VL、Ovis)中广泛应用；而相对位置编码和视觉 RoPE则代表了更具理论优势的前沿方向。

### 2. 残差连接的核心作用
#### 1. 缓解梯度消失问题
(1)在深度网络中，反向传播时梯度需逐层相乘，若权重矩阵谱范数小于1，梯度会指数级衰减；(2)残差连接提供了一条恒等映射路径(identity path)，使得梯度可直接回传至浅层，避免完全依赖非线性变换路径；(3)数学上，总梯度包含两项：来自子层的梯度 + 来自残差路径的单位梯度，后者保证了梯度下界，显著提升训练稳定性。

### 2. 促进信息流动与特征复用
(1)低层捕获的细粒度特征(如边缘、纹理)可通过残差连接直接传递至高层；(2)高层无需从零重建这些基础信息，而是在其基础上叠加抽象语义(如物体、关系)；(3)这种“增量式学习”机制使模型能更高效地利用各层表示，避免信息在深层被覆盖或丢失。

### 3. 支持更深网络的训练
(1)原始 Transformer 论文使用 6 层编码器-解码器，而现代大模型(如 LLaMA、GPT)可达数十甚至上百层；(2)若无残差连接，如此深度的网络几乎无法收敛；(3)实验证明，移除残差连接后，即使使用先进优化器，模型在 few-layer 情况下也会出现训练震荡或发散。

### 3. 残差连接与其他组件的协同
#### 1. 与 Layer Normalization 的配合
(1)残差连接后通常紧跟 LayerNorm，用于稳定激活值分布；(2)Post-LN 结构中，LayerNorm 作用于残差和子层输出之和，有助于控制梯度尺度；(3)Pre-LN 结构(如 T5、LLaMA)将 LayerNorm 移至子层输入端，进一步提升训练稳定性，尤其适用于超深模型。

#### 2. 对模型容量的影响
(1)残差连接不增加参数量，但有效扩展了函数空间；(2)理论上，带残差的网络可学习恒等映射(当子层输出为0时)，而普通网络难以做到；(3)这使得模型在必要时“跳过”某些层，实现自适应深度，提升表达灵活性。

### 3. 在大模型中的实际表现
(1)在百亿级以上参数模型中，残差连接是维持训练动态平衡的关键；(2)若初始化不当或残差路径断裂(如错误实现)，常导致 loss 震荡或 NaN；(3)工程实践中，常通过梯度检查、激活值监控等手段验证残差路径是否正常工作。

### 4. 常见误解与注意事项
#### 1. 残差连接 ≠ 跳跃连接(Skip Connection)
(1)跳跃连接是更广义的概念，残差连接是其特例(输出 = 输入 + F(输入))；(2)在 ResNet 中首次系统化应用，后被 Transformer 继承并成为标配。

#### 2. 不能随意移除或修改
(1)有研究尝试用门控机制替代残差连接，但通常带来性能下降或训练复杂度上升；(2)在模型压缩或硬件部署中，必须保留残差加法操作，否则破坏训练-推理一致性。

#### 3. 与 Attention 机制的互补性
(1)Attention 提供动态上下文聚合，残差连接保障信息保真传输；(2)二者共同构成 Transformer “灵活建模 + 稳定训练”的双支柱。

残差连接虽看似简单，却是 Transformer 能够扩展至超大规模并保持稳定训练的基石之一。它通过引入恒等路径，从根本上缓解了深度网络的优化难题，同时促进了跨层信息的有效复用。

 一、Focal Loss 的提出背景与核心动机

 Focal Loss 最初由 Lin 等人在 2017 年的论文《Focal Loss for Dense Object Detection》中提出，旨在解决 **目标检测任务中极端前景-背景类别不平衡问题**。

 在单阶段检测器(如 SSD、YOLO)中，每个图像可能生成数万甚至数十万个候选框(anchors)，但其中绝大多数是负样本(背景)，正样本(目标)占比常低于 0.1%。这种极度不平衡会导致模型训练被大量简单负样本主导，难以有效学习稀有但关键的正样本特征。

 标准交叉熵损失(Cross-Entropy Loss)对所有样本一视同仁，而 Focal Loss 通过引入 **动态缩放因子**，自动降低易分类样本的权重，使模型聚焦于难分类样本，从而提升整体性能。

 三、Focal Loss 的特性与优势
 1. 自适应样本加权
 (1)无需显式采样(如 OHEM)或硬阈值筛选，自动根据模型当前表现动态调整每个样本的贡献；
 (2)避免了采样带来的信息丢失或训练不稳定问题。
 2. 梯度行为分析
 (1)对易分样本，梯度幅值被显著压缩，减少其对参数更新的影响；
 (2)对难分样本，梯度接近原始交叉熵，确保有效学习；
 (3)整体优化方向更聚焦于提升模型判别边界附近的样本。
 3. 适用场景扩展
 (1)虽源于目标检测，但已广泛应用于其他类别不平衡任务：
 1. 医学图像分割(病灶区域占比极小)；
 2. 异常检测(正常样本远多于异常)；
 3. 长尾分类(少数类别样本稀少)；
 4. 多标签文本分类(正标签稀疏)。

 四、在大模型中的应用与变体
 1. 多标签分类中的使用
 (1)大语言模型或视觉-语言模型在处理多标签输出时(如图像标签预测)，常面临标签稀疏问题；
 (2)Focal Loss 可替代 BCEWithLogitsLoss，提升对稀有标签的召回率。
 2. 变体与改进
 (1) **Quality Focal Loss** (QFL)：将标签软化为质量分数(如 IoU)，用于密集检测中的定位质量估计；
 (2) **Distribution Focal Loss** (DFL)：用于回归任务，将连续值建模为离散分布；
 (3) **Asymmetric Focal Loss**：对正负样本使用不同 γ，适用于正样本极其稀疏的场景。
 3. 与大模型训练的结合
 (1)在指令微调或多任务学习中，若某些任务数据量极少，可对其损失项采用 Focal 形式；
 (2)在强化学习奖励塑形中，也可借鉴其“聚焦难样本”思想设计优势函数。

 五、总结

 Focal Loss 通过一个简洁而优雅的数学形式，解决了机器学习中长期存在的类别不平衡难题。其核心思想—— **降低易分样本权重，聚焦难例学习** ——不仅提升了目标检测性能，更启发了后续众多损失函数的设计。

 在大模型时代，尽管预训练数据规模庞大，但在特定下游任务(如细粒度识别、罕见事件检测)中，Focal Loss 仍是应对数据偏态分布的有效工具。

 一、PPO的基本定位与应用场景

 PPO(Proximal Policy Optimization)是一种策略梯度类的强化学习算法，因其 **稳定性高、实现简单、样本效率适中**，成为大语言模型对齐阶段(如 RLHF)的主流优化方法。在 LLM 训练中，PPO 用于微调一个初始策略(通常为 SFT 模型)，使其生成结果更符合人类偏好或特定奖励信号，同时避免策略更新过大导致性能崩溃。

 二、PPO训练的整体流程
 1. 前置准备阶段
 (1) **监督微调模型** (SFT Model)：作为初始策略，提供合理的行为先验；
 (2) **奖励模型** (Reward Model, RM)：由人类偏好数据训练得到，输入 prompt + response，输出标量奖励；
 (3) **环境模拟**：将 LLM 自身作为环境，给定 prompt 后采样生成 response，再由 RM 打分。
 2. PPO迭代训练循环
 (1) **数据采样** (Rollout)：
 1. 从 prompt 数据集随机采样一批 prompts；
 2. 使用当前策略 对每个 prompt 生成一个或多个 responses；
 3. 调用 RM 计算每个 response 的奖励；
 4. 同时记录生成过程中的 log-probabilities(用于后续重要性采样)。
 (2) **优势函数估计** (Advantage Estimation)：
 1. 在 LLM 场景中，通常采用 **reward 直接作为优势** (因无状态转移，单步回报即为总回报)；
 2. 更严谨的做法是减去基线(baseline)，如该 prompt 下历史平均奖励，以降低方差；
 3. 最终得到每个 token 或整个序列的优势值。
 (3) **策略更新** (Policy Update)：
 1. 构建 PPO 目标函数(见下文)；
 2. 使用 mini-batch SGD(如 Adam)多次更新策略网络；
 3. 每轮更新后，将当前策略设为“旧策略”，用于下一轮采样。

 三、PPO的核心机制：目标函数设计
 1. 重要性采样比率
 (1)定义新旧策略在某 action 上的概率比：
 (2)该比率衡量新策略相对于旧策略的“偏离程度”；
 (3)直接最大化 是策略梯度的目标，但易导致大步更新。

 四、PPO在大模型训练中的工程实践
 1. 采样与奖励计算优化
 (1)使用 beam search 或 top-p 采样生成高质量候选，而非纯随机；
 (2)对同一 prompt 生成多个 responses，构建 pairwise 偏好，提升 RM 信噪比；
 (3)缓存 RM 打分结果，避免重复调用。
 2. 训练稳定性技巧
 (1) **小学习率**：通常设为 量级，避免破坏预训练知识；
 (2) **多 epoch 更新**：对同一批 rollout 数据进行 2–4 次策略更新，提高样本利用率；
 (3) **梯度裁剪**：限制梯度范数，防止爆炸；
 (4) **监控指标**：跟踪 reward、KL 散度、policy loss 等，及时发现发散。
 3. 与替代方法的对比
 (1)相比 TRPO，PPO 无需二阶优化，实现更简单；
 (2)相比 vanilla policy gradient，PPO 通过裁剪显著提升稳定性；
 (3)近年出现的 DPO(Direct Preference Optimization)绕过 PPO 和 RM，直接从偏好数据优化策略，训练更高效，但 PPO 仍因其通用性和可控性被广泛使用。

 五、总结

 PPO 通过引入裁剪机制，在策略更新的激进性与保守性之间取得平衡，使其成为大模型强化学习对齐阶段的可靠选择。

 尽管其训练流程涉及采样、奖励建模、优势估计和策略优化等多个环节，但核心思想清晰： **在不大幅偏离原策略的前提下，朝着高奖励方向小幅改进**。

## 0x06. 大模型算法：zero1、2、3原理
> 发布日期：2026-05-27  
> 原文链接：[大模型算法：zero1、2、3原理](https://mp.weixin.qq.com/s/1N26d-FK97nNiUaIqRZwlw)
### 1. ZeRO 的核心思想与背景
DeepSpeed 提出的Zero Redundancy Optimizer(ZeRO) 是一种面向超大规模模型训练的显存优化技术，旨在消除传统数据并行中各 GPU 上重复存储的优化器状态、梯度和模型参数。

通过将这些组件在设备间进行切分(partitioning)，ZeRO 显著降低单卡显存占用，使得百亿甚至万亿参数模型可在普通 GPU 集群上训练。ZeRO 分为三个阶段(Stage 1/2/3)，逐级切分更多组件，形成递进式优化体系。

### 2. ZeRO-1：优化器状态切分
1. 切分对象(1)仅对优化器状态(如 Adam 中的 momentum 和 variance)进行切分；(2)每个 GPU 仅存储总优化器状态的 1/N(N 为 GPU 数量)；(3)模型参数和梯度仍完整保留在每张卡上。
2. 工作流程(1)前向和反向计算与标准数据并行一致，各 GPU 独立完成；(2)反向结束后，各 GPU 拥有完整的梯度；(3)在参数更新阶段，每个 GPU 仅使用自己负责的那部分梯度，结合本地存储的优化器状态，更新对应的参数子集；(4)更新完成后，通过 all-gather 操作同步所有参数，确保模型一致性。
3. 显存节省效果(1)以 Adam 优化器为例，其状态大小约为参数量的 2 倍(FP32)；(2)ZeRO-1 可将优化器状态显存降至 1/N，适用于中小规模模型(如 10B 以下)；(3)通信开销与标准数据并行相当，仅增加一次 all-gather。

### 3. ZeRO-2：梯度 + 优化器状态切分
1. 切分对象扩展(1)在 ZeRO-1 基础上，进一步将梯度也进行切分；(2)反向传播完成后，各 GPU 仅保留对应参数块的梯度；(3)其余梯度在 reduce-scatter 操作后立即释放。
2. 工作流程优化(1)反向过程中，梯度通过 reduce-scatter 聚合到对应 GPU(而非 all-reduce)；(2)每个 GPU 直接在其本地梯度和优化器状态上执行参数更新，无需等待全局梯度；(3)更新后仍需 all-gather 同步参数。
3. 显存与性能优势(1)梯度和优化器状态显存均降至 1/N，总节省约 2–4 倍于 ZeRO-1；(2)通信总量不变(仍为 O(P))，但通信模式从 all-reduce 改为 reduce-scatter + all-gather，更利于带宽利用；(3)是当前最广泛使用的配置，平衡效率与实现复杂度。

### 4. ZeRO-3：参数 + 梯度 + 优化器状态全切分
1. 切分对象全覆盖(1)进一步将模型参数本身也切分到各 GPU；(2)每张卡仅持有 1/N 的参数、梯度和优化器状态；(3)显存占用理论上降至传统数据并行的 1/N。
2. 动态参数收集机制(1)前向计算时，当前层所需参数若不在本地，则通过 all-gather 临时获取；(2)计算完成后立即释放，避免长期驻留；(3)引入参数预取(prefetching)和持久化缓存(persistent cache)策略，隐藏通信延迟。
3. 关键技术挑战与应对(1)通信频率大幅增加，对网络带宽要求高；(2)需精细调度计算与通信流水线，避免 GPU 空闲；(3)通常与激活值检查点(activation checkpointing)和混合精度训练结合使用。
4. 适用场景(1)支持千亿至万亿参数模型训练(如 Megatron-Turing NLG)；(2)在千卡集群上可将单卡显存需求从数百 GB 降至 10–20 GB；(3)常与模型并行(如张量并行)组合，形成 3D 并行策略。

### 5. 三阶段对比与工程权衡
1. 显存节省对比(以 100B 模型、128 GPU 为例)(1)ZeRO-1：仅优化器状态切分，显存仍受参数和梯度限制；(2)ZeRO-2：梯度+优化器状态切分，单卡显存降至 ~5 GB；(3)ZeRO-3：全切分，单卡显存可控制在 ~6 GB 以内。

#### 2. 通信开销趋势(1)ZeRO-1/2：通信量与模型大小成正比，适合高带宽网络；(2)ZeRO-3：通信次数显著增加，但每次数据量小，需低延迟网络支持。
3. 实现复杂度(1)ZeRO-1/2：集成于 DeepSpeed，API 简单，易于部署；(2)ZeRO-3：需处理动态参数加载，调试难度高，通常用于超大规模训练。

ZeRO-1、ZeRO-2、ZeRO-3 构成了一个渐进式的显存优化体系，通过逐级切分优化器状态、梯度和模型参数，在不改变模型结构的前提下，将大模型训练从“显存受限”转向“通信受限”。其中，ZeRO-2 因其良好的性价比成为工业界主流选择，而 ZeRO-3 则支撑了前沿万亿级模型的训练。

## 0x07. 大模型算法：什么是排序模型
> 发布日期：2026-05-27  
> 原文链接：[大模型算法：什么是排序模型](https://mp.weixin.qq.com/s/j3cbGF6ASkCd8WQKVZ49NA)
### 1. 排序模型的基本定义与目标
排序模型(Ranking Model)是一类用于对一组候选项目(items)按照其与给定查询(query)的相关性或用户偏好进行相对排序的机器学习模型。其核心目标不是预测绝对分数，而是学习一个排序函数，使得更相关或更受偏好的项目排在前面。

在大模型时代，排序模型广泛应用于搜索引擎、推荐系统、广告投放、对话系统回复选择等场景。例如，在问答系统中，若模型生成多个候选回答，排序模型可从中选出最符合人类偏好的一个；在检索增强生成(RAG)中，排序模型对召回的文档片段按相关性重排，提升上下文质量。

### 2. 排序模型的学习范式
#### 1. 点级排序(Pointwise)
(1)将排序问题转化为对每个“查询-项目”对的独立评分任务；(2)标签可以是二值(相关/不相关)或多级(1–5 星)；(3)常用损失函数包括交叉熵(分类)或均方误差(回归)；(4)优点是实现简单，可复用分类/回归模型；缺点是忽略项目间的相对关系。

### 2. 对级排序(Pairwise)
(1)关注任意两个项目的相对顺序：若项目 A 比 B 更相关，则模型应给出更高分；(2)训练样本为三元组(query, item_A, item_B)及偏好标签；(3)典型算法包括 RankNet、RankSVM；(4)损失函数常基于 sigmoid 或 hinge loss，鼓励正确序对的得分差大于 margin；(5)比点级更贴近排序本质，但组合数随项目数平方增长，计算开销大。

### 3. 列表级排序(Listwise)
(1)直接优化整个排序列表的评价指标(如 NDCG、MAP)；(2)代表方法包括 ListNet、LambdaRank、LambdaMART；(3)通过梯度近似(如 LambdaLoss)将不可导的排序指标转化为可学习信号；(4)性能通常优于前两类，但实现复杂，对噪声敏感。

### 3. 大模型时代的排序模型演进
#### 1. 从传统模型到大语言模型
作为排序器
(1)早期排序模型多基于浅层模型(如 LR、GBDT)或浅层神经网络；(2)随着 LLM 发展，其强大的语义理解能力使其天然适合作为排序函数；(3)例如，在 RAG 系统中，LLM 可接收“查询 + 多个文档片段”，输出每个片段的相关性判断或直接重排。

#### 2. 基于偏好的排序：RLHF 中的奖励模型
(1)在强化学习人类反馈(RLHF)流程中，奖励模型(Reward Model, RM)本质上是一个排序模型；(2)它接受同一 prompt 下的多个模型输出，并学习人类标注的偏好顺序；(3)RM 通常基于 LLM 微调，使用 pairwise ranking loss(如 Bradley-Terry 模型)；(4)训练后的 RM 为 PPO 提供奖励信号，引导策略模型生成更优回答。

### 3. 高效排序架构设计
(1)为降低推理成本，实践中常采用“两阶段”排序：1. 第一阶段：轻量模型(如 BM25、双塔 DSSM)粗排，召回数百候选；2. 第二阶段：大模型精排，对 top-K 候选进行细粒度打分；(2)也可使用 Cross-Encoder 架构(如 BERT-based reranker)，将 query 和 item 拼接输入，获得高精度相关性分数；(3)近期工作探索用小模型蒸馏大模型排序能力，实现效果与效率平衡。

### 4. 评估指标与挑战
#### 1. 常用评估指标
(1)NDCG@K(Normalized Discounted Cumulative Gain)：衡量 top-K 排序质量，考虑位置衰减；(2)MAP(Mean Average Precision)：适用于二值相关性标签；(3)MRR(Mean Reciprocal Rank)：关注首个相关结果的位置；(4)AUC：在 pairwise 设置下评估模型区分正负序对的能力。

#### 2. 实际挑战
(1)标注成本高：高质量排序标签需人工比较，尤其在 open-ended 任务中；(2)分布偏移：训练数据中的排序偏好可能与线上用户行为不一致；(3)公平性与多样性：过度优化相关性可能导致信息茧房或群体偏差；(4)延迟约束：大模型排序虽准，但响应时间可能不满足实时系统要求。

排序模型是连接信息检索、推荐系统与大语言模型的关键桥梁。其核心在于学习“相对优劣”而非“绝对真值”。在大模型背景下，排序任务既受益于 LLM 的强大语义能力，也面临效率、标注和对齐等新挑战。

无论是作为 RLHF 中的奖励模型，还是 RAG 中的重排器，排序模型都在确保大模型输出更相关、更安全、更符合用户意图方面发挥着不可替代的作用。

## 0x08. 大模型算法：排序模型有哪些
> 发布日期：2026-05-27  
> 原文链接：[大模型算法：排序模型有哪些](https://mp.weixin.qq.com/s/wHn-ICS05LbX9ZXR2x27yQ)
### 1. 排序模型的基本概念与应用场景
排序模型(Learning to Rank, LTR)是一类用于对候选集合按照相关性、质量或用户偏好进行有序排列的机器学习模型。其核心目标不是预测绝对分数，而是学习一个相对顺序关系。这类模型广泛应用于：
1. 搜索引擎：对网页结果按查询相关性排序；2、推荐系统：对商品、视频、新闻等按用户兴趣排序；3、广告系统：对广告按点击率或转化率排序；4、大模型输出控制：对生成候选按多样性、安全性或多维偏好排序。

在大模型时代，排序模型既可作为独立模块(如重排器)，也可嵌入生成流程(如对比解码、奖励引导采样)。

### 2. 排序模型的主要类别
#### 1. Pointwise 方法
(1)基本思想：将排序问题转化为单个样本的回归或分类任务，每个 item 独立打分，最终按分数排序；(2)典型模型：1. Logistic Regression(LR)：预测点击概率；2. Gradient Boosting Decision Trees(GBDT，如 XGBoost、LightGBM)：处理高维稀疏特征；3. Deep Neural Networks(DNN)：如 YouTube DNN 推荐模型；(3)优点：实现简单，可复用成熟分类/回归框架；(4)缺点：忽略 item 之间的相对关系，排序性能受限。

### 2. Pairwise 方法
(1)基本思想：将排序转化为成对比较问题，学习“item A 是否比 item B 更相关”；(2)典型模型：1. RankNet：使用神经网络建模 pair 概率，采用交叉熵损失；2. RankSVM：基于 SVM 的 pairwise 排序；3. GBRank：GBDT 的 pairwise 扩展；(3)损失函数通常基于 Bradley-Terry 模型：
(4)优点：显式建模相对顺序，效果优于 pointwise；(5)缺点：训练复杂度高(O(n²) pairs)，且未直接优化最终排序指标(如 NDCG)。

### 3. Listwise 方法
(1)基本思想：直接以整个候选列表为单位进行优化，目标函数与实际排序评价指标对齐；(2)典型模型：1. ListNet：最小化预测分布与真实分布的交叉熵；2. ListMLE：最大化正确排序的似然；3. SoftRank / ApproxNDCG：对 NDCG 等不可导指标进行平滑近似，支持梯度下降；(3)优点：直接优化业务指标，理论性能最优；(4)缺点：计算开销大，实现复杂，对噪声敏感。

### 3. 大模型时代的排序模型新范式
#### 1. 基于大语言模型的排序(LLM-based Reranking)
(1)利用 LLM 强大的语义理解能力，对候选进行上下文感知重排；(2)典型方法：1.Prompt-based Ranking：构造 prompt 如 “Which is more relevant to ‘AI’? A: ‘machine learning’ B: ‘coffee’”，让 LLM 选择；2.Score-based Ranking：让 LLM 为每个候选生成相关性分数(如 1–5 分)；3.List-wise Generation：直接让 LLM 输出排序后的列表(如 “Top-3 most relevant: ...”)；(3)优势：无需标注数据，zero-shot 能力强；(4)挑战：推理成本高，输出不稳定，需后处理校验。

#### 2. 偏好模型(Preference Model)
作为排序器
(1)在 RLHF 或 DPO 流程中训练的 Reward Model(RM)本质是一个 pairwise 排序模型；(2)输入为 (prompt, response_A, response_B)，输出为偏好概率；(3)可用于对多个生成候选进行两两比较，构建全序(如通过排序算法如 QuickSort)；(4)代表工作：Anthropic 的 HH 数据集训练的 RM、OpenAI 的 GPT-4 评分模型。

### 3. 多维度排序与约束排序
(1)现代系统常需同时优化多个目标(如相关性、多样性、新鲜度、公平性)；(2)方法包括：1. 多任务学习：共享底层表示，多个排序头；2. 加权融合：对各维度打分加权求和；3. Pareto 优化：寻找非支配解集；(3)大模型可通过指令控制排序倾向，例如 “请按新颖性而非流行度排序”。

### 4. 评估排序模型的常用指标
1. MAP(Mean Average Precision)：衡量排序中相关 item 的平均精度；2、NDCG(Normalized Discounted Cumulative Gain)：考虑位置折损的相关性累积得分；3、MRR(Mean Reciprocal Rank)：关注首个相关 item 的排名倒数；4、AUC(Area Under ROC Curve)：适用于 pairwise 模型的判别能力评估。

排序模型从早期的 Pointwise 回归，发展到 Pairwise 比较，再到 Listwise 全局优化，体现了对排序本质理解的深化。

在大模型时代，排序不再局限于传统特征工程与浅层模型，而是与语言理解、偏好学习、多目标优化深度融合。

无论是作为独立的重排模块，还是内嵌于生成控制流程，排序模型依然是连接模型能力与用户体验的关键桥梁。
