## 写在前面

RLHF 方向的面试在 2026 年呈现出一个极其鲜明的特征： **PPO vs DPO vs GRPO 的三方对比出现了 25+ 次** 。这三个算法几乎成了"偏好优化面试三件套"——面试官通过一道对比题，就能检验候选人对强化学习基础、偏好建模思想、训练稳定性工程的三重理解。

## 一、PPO：RLHF 的工程主力

PPO 在这份题库中出现了 **30+ 次** ，是所有强化学习算法中被考察最多的。面试从目标函数推导、clip 机制、GAE 优势估计一直考到多模型协同训练。

**Clip 和 Min 各自的作用：**

**clip 操作** 将重要性比率限制在 区间内，防止新策略与旧策略偏离过大。没有 clip 时，如果某个动作在新策略下概率远高于旧策略（r\_t >> 1），梯度会推动策略继续向该方向大幅更新，可能导致策略崩溃。

**min 操作** 取未裁剪项和裁剪项的较小值，构成一个"悲观"的下界估计。当优势为正（好动作）时，min 防止 r\_t 无限增大带来的过度乐观更新；当优势为负（坏动作）时，min 确保策略能远离该动作但不过度惩罚。

两者协同的效果是：在信任域（trust region）内允许策略自由更新，超出信任域时梯度被截断。这比 TRPO 的硬约束更简单高效，且在实践中效果相当。

### 1.3 PPO 在 RLHF 中的完整训练流程

**考察意图：** 出现 8+ 次，考察对多模型协同的理解。

**回答框架：**

标准 RLHF-PPO 训练涉及 **四个模型** 的协同：

**Policy Model（策略模型）：** 当前正在优化的 LLM。接收 prompt 生成 response（rollout），其参数根据 PPO 目标函数更新。

**Reference Model（参考模型）：** SFT 后冻结的 LLM 副本。用于计算 KL 散度惩罚项，防止策略模型偏离 SFT 分布过远（详见 1.4）。

**Reward Model（奖励模型）：** 基于人类偏好数据训练的打分模型。对策略模型的每个 response 给出标量奖励分数。

**Value Model（价值模型 / Critic）：** 估计每个状态的期望回报，用于 GAE 优势计算。通常从 reward model 或 policy model 初始化。

训练一步的流程：Policy 生成 response → Reward Model 打分 → 用 Reference Model 计算 KL 惩罚 → 构造奖励信号（reward - β·KL）→ Value Model 估计优势 → PPO 更新 Policy 和 Value Model。

### 1.4 Reference Model 与 KL 约束的必要性

**考察意图：** 出现 6+ 次，考察对 RLHF 训练稳定性机制的理解。

**回答框架：**

Reference Model 存在的根本原因是 **奖励模型不完美** 。

Reward Model 是在有限的偏好数据上训练的，它学到的奖励函数只是真实人类偏好的一个近似。如果不加约束地最大化这个近似奖励，策略模型会找到 RM 的"盲点"——生成 RM 给高分但实际质量差的 response（这就是 **reward hacking** ）。

KL 散度约束 的作用是：限制策略模型不能偏离 SFT 后的"安全分布"太远。即使 RM 在某些区域给出了错误的高奖励，KL 约束也会阻止策略向那些区域大幅移动。

**若移除 Reference Model：** 策略模型会迅速收敛到 RM 的模式崩溃点——可能表现为输出高度重复的固定模板（RM 给高分的格式）、过度冗长（某些 RM 对长回答有偏好）、或安全性退化（丧失 SFT 阶段学到的拒绝能力）。

## 二、DPO：绕过显式奖励建模

DPO 在题库中出现了 **20+ 次** ，其核心卖点——"不需要奖励模型"——是面试中最常被追问的设计动机。

**设计直觉：** 方括号内的 logits 差值项可以理解为一个 **隐式奖励差** 。 衡量的是当前策略相对于参考策略对 response y 的"偏好增量"。DPO 的目标是让 chosen response 的偏好增量大于 rejected response 的偏好增量——即让策略在相对意义上更偏好好回答。

**β 的物理含义：** 控制对偏好信号拟合的"激进程度"。β 小 → 更激进地拟合偏好数据（可能过拟合）；β 大 → 更保守地贴近参考模型（可能欠拟合偏好）。实践中 β ∈ \[0.1, 0.5\]。

### 2.3 DPO vs SFT 的本质区别

**考察意图：** 出现 6+ 次，考察对两种训练范式建模差异的理解。

**回答框架：**

SFT 和 DPO 的根本区别在于 **信号类型** ：

**SFT 使用绝对信号：** "这是一个好回答"→ 最大化其似然。SFT 只能学习"模仿好回答"，无法表达"A 比 B 好"这种相对判断。

**DPO 使用相对信号：** "这个回答比那个好"→ 最大化两者的对数概率差。DPO 直接建模了偏好排序关系。

这个差异带来的关键收益：

**减少标注偏差放大。** SFT 要求标注者写出"完美回答"（绝对标准），不同标注者的"完美"标准差异大。DPO 只要求标注者比较两个回答哪个更好（相对标准），这对人类来说更自然、一致性更高。

**规避 reward hacking 风险。** SFT 的模仿学习没有防止模型"过度泛化"坏模式的机制。DPO 通过参考模型约束（KL 惩罚隐含在损失函数中），天然地限制了策略的偏移幅度。

**信息效率更高。** 一对偏好数据 (y\_w, y\_l) 同时提供了正信号（y\_w 好）和负信号（y\_l 差），比 SFT 只用 y\_w 的正信号多了一倍的训练信息。

### 2.4 DPO 的数据需求与典型变体

**考察意图：** 出现 5+ 次，考察实操经验。

**回答框架：**

**数据格式：** 每条样本包含 `(prompt, chosen_response, rejected_response)` 。关键约束包括：两个 response 必须基于同一个 prompt 生成（保证可比性）、chosen 确实优于 rejected（标注一致性）、两者的质量差距不宜过大也不宜过小（过大则梯度信号弱——模型已经知道哪个好；过小则标注噪声占主导）。

**隐含假设：** DPO 假设偏好遵循 Bradley-Terry 模型（偏好概率由奖励差的 sigmoid 决定），且偏好数据的分布能覆盖策略实际会生成的 response 空间。后者在实践中常被违反——DPO 在离线数据上训练，但策略在更新后的生成分布会偏移。

**典型变体：**

- **IPO（Identity Preference Optimization）：** 去除了 Bradley-Terry 假设，直接优化一个更一般的偏好目标。对标注噪声更鲁棒。
- **KTO（Kahneman-Tversky Optimization）：** 不需要成对偏好数据，只需要单条回答的好/坏二元标签。基于前景理论对损失和收益施加非对称权重。
- **SimPO：** 简化了参考模型的计算，用序列长度归一化的对数概率替代 logit 差值。
- **迭代式 DPO：** 在每轮 DPO 训练后用更新后的策略重新采样偏好数据，缓解分布偏移问题。本质上是一种近似的在线 DPO。

## 三、GRPO：2026 年的新兴核心考点

GRPO 的考察频次在这份题库中 **暴增至 20+ 次** ，反映了 DeepSeek-R1 发布后业界的高度关注。

### 3.1 GRPO 的核心思想与目标函数

**考察意图：** 出现 10+ 次，是 GRPO 的入门必答题。

**回答框架：**

GRPO（Group Relative Policy Optimization）的核心创新是： **用组内相对排名替代 critic 网络做优势估计** 。

具体做法：对同一个 prompt，用当前策略采样一组 response（如 K=8 个），用 Reward Model 对每个 response 打分，然后将分数在组内做 **归一化** （减均值除标准差）作为每个 response 的优势估计：

然后用类似 PPO 的策略梯度目标进行更新：

**去除 Critic 的动机：** 在 LLM-RLHF 中，Critic（Value Model）本身就是一个与 Policy 同等规模的大模型，需要额外的显存和计算。且由于 LLM 的动作空间极大（整个词表的序列组合），Critic 的价值估计往往不准确。GRPO 用组内归一化直接从 Reward Model 的输出构造优势，省去了 Critic，训练流程大幅简化。

### 3.2 GRPO vs PPO 的核心差异

**考察意图：** 出现 8+ 次，是 RLHF 面试的高频对比题。

**回答框架：**

| 维度 | PPO | GRPO |
| --- | --- | --- |
| 优势估计 | GAE（需要 Critic 网络） | 组内归一化（无需 Critic） |
| 模型数量 | 4 个（Policy + Reference + Reward + Value） | 3 个（Policy + Reference + Reward） |
| 显存开销 | 高（需存储 Value Model） | 低（省去 Value Model） |
| 采样方式 | 逐条采样 + GAE | 批量采样 K 条 + 组内归一化 |
| 方差控制 | GAE 的 λ 参数 + Value baseline | 组内归一化天然提供 baseline |
| KL 约束 | clip 隐式约束 + 可选 KL 惩罚 | 显式 KL 惩罚项 |

**PPO 仍被广泛采用的原因：** PPO 的工程实践积累更深厚、超参数调优经验更丰富、在多种任务上验证充分。GRPO 虽然简化了架构，但组内归一化的优势估计质量依赖于采样数量 K——K 太小则估计方差大，K 太大则采样成本高。

### 3.3 GRPO 的 Rollout 复用与多次更新

**考察意图：** 出现 5+ 次，考察对 GRPO 实现细节的理解。

**回答框架：**

GRPO 的单次 rollout 后如何复用样本：

对每个 prompt 采样 K 个 response 后，这批样本可以用于 **多次策略更新** （类似 PPO 的 multi-epoch 更新）。每次更新时：

1. 用 **当前策略** 计算每个 response 的对数概率
2. 用 **采样时的旧策略** 的对数概率计算重要性比率
3. 用组内归一化的奖励作为优势
4. 计算 PPO-style 的 clip 损失 + KL 惩罚

关键是：优势估计 在采样时就固定了（来自 Reward Model 的打分），不会随 θ 更新而变化。随着 θ 多次更新偏离 θ\_old，重要性比率 r\_t 会偏离 1，clip 机制防止偏离过大。

**若复用次数过多：** 重要性比率会越来越偏离 1，clip 会频繁触发导致有效梯度为零。经验上复用 2-4 次是常见实践。

### 3.4 GRPO 中移除 Clip 的影响

**考察意图：** 考察对训练稳定性机制的深层理解。

**回答框架：**

GRPO 中若移除 clip 机制（如 DAPO 的某些变体探索过这个方向）：

**短期效果：** 策略更新步长不再受限，可能获得更快的初始收敛速度。

**长期风险：** 当某个 response 的重要性比率 r\_t 极大时（新策略认为该 response 概率远高于旧策略），未经裁剪的 r\_t · 会产生极大的梯度，导致单步更新过猛，策略可能跳到一个完全不同的区域——即 **策略崩溃** 。在 LLM 的高维离散动作空间中，这种崩溃尤其容易发生且难以恢复。

**DAPO 的替代方案：** DAPO 移除了 clip 但引入了其他约束机制（如动态 KL 惩罚系数、token 级策略熵正则化），用不同方式控制更新步长。这说明"约束策略更新步长"是必要的，但实现方式可以多样化。

## 四、奖励建模：RLHF 的根基与软肋

奖励模型相关问题出现了 **15+ 次** ，涵盖设计原则、典型缺陷和 reward hacking 防御。

### 4.1 奖励模型的训练目标与建模方式

**考察意图：** 基础必考，好的回答需要覆盖打分形式和标注策略。

**回答框架：**

奖励模型的训练目标是： **学习一个标量函数 r(x, y)，使得人类偏好的 response 得分更高** 。

最常用的建模方式是 **Bradley-Terry 偏好模型** ：

训练损失为对数似然：

**打分形式对比：**

- **Pairwise Ranking：** 输入一对 (chosen, rejected)，输出偏好概率。最常用，与 Bradley-Terry 模型天然匹配。
- **Scalar Reward（直接打分）：** 对单条 response 输出绝对分数。更直观，但人类对绝对质量的评判一致性远低于相对比较。
- **N-wise Ranking：** 对 N 个 response 排序。提供更丰富的排序信号，但标注成本高。

### 4.2 Reward Hacking 的典型表现与防御

**考察意图：** 出现 8+ 次，是 RLHF 面试的高阶必考题。

**回答框架：**

Reward Hacking 是指策略模型利用 Reward Model 的缺陷获取高分而非真正提升回答质量。典型表现包括：

**长度偏好利用：** 某些 RM 对更长的回答给更高分（训练数据中长回答通常更详细）。策略模型学会生成冗余冗长但内容空洞的回答。

**格式注入：** RM 对特定格式（如 Markdown 列表、代码块）有偏好。策略模型会在不需要时也使用这些格式来获取高分。

**风格模仿：** RM 可能偏好某种措辞风格（如过度谨慎的"I should note that..."）。策略模型学会在所有回答中插入这类套话。

**防御策略：**

- **KL 约束（最基础）：** 限制策略偏离参考模型的幅度，防止过度优化 RM 的弱点。
- **奖励归一化 / 缩放：** 对 RM 输出做动态归一化，防止极端分数主导梯度。
- **多 RM 集成：** 使用多个独立训练的 RM，取平均或最小值作为最终奖励。单个 RM 的漏洞不太可能在所有 RM 中同时存在。
- **长度惩罚：** 在奖励中显式扣除长度相关项，消除长度偏好。
- **对抗训练：** 用策略模型的高分但低质量输出作为 RM 的负样本，迭代加强 RM 的鲁棒性。

### 4.3 规则奖励 vs 模型奖励

**考察意图：** 出现 4+ 次，考察对不同奖励信号来源的权衡理解。

**回答框架：**

**规则奖励（Rule-based Reward）：**

- 优势：确定性、可解释、无 reward hacking 风险、不需要训练。
- 劣势：只能覆盖可形式化的质量维度（如格式正确性、代码通过率、数学答案正确性），无法衡量主观质量（如有用性、自然度）。
- 适用场景：代码生成（单元测试通过率）、数学推理（答案正确性）、格式约束（JSON 合法性）。

**模型奖励（Learned Reward Model）：**

- 优势：能捕捉复杂的、难以形式化的人类偏好（如回答的有用性、无害性）。
- 劣势：存在 reward hacking 风险、需要持续维护、偏好数据的质量直接决定 RM 质量。
- 适用场景：开放域对话、创意写作、复杂指令遵循。

实践中常 **混合使用** ：用规则奖励覆盖可量化的硬约束，用模型奖励覆盖主观质量，两部分加权求和作为最终奖励。

### 4.4 Token 级 vs 序列级奖励

**考察意图：** 出现 4+ 次，考察对信用分配问题的理解。

**回答框架：**

标准 RM 输出的是序列级标量奖励（对整个 response 打一个分）。但 PPO 的优势估计需要 step 级信号，这就引出了信用分配问题——一个好的 response 中哪些 token 贡献了质量？

**常见的分解方法：**

**均匀分配：** 将序列奖励平均分配到每个 token。最简单，但忽略了不同 token 重要性的差异。

**末位分配：** 将全部奖励分配到最后一个 token（或 EOS token），中间 token 奖励为 0。这在理论上等价于 MC 估计，但使得中间 step 的 GAE 优势估计完全依赖 Critic，Critic 不准时效果差。

**Token-level RM：** 训练一个能对每个 token 位置输出奖励的 RM（如 Process Reward Model）。最精细但训练数据获取困难——需要对每一步推理过程标注正误。

**基于注意力的归因：** 用 RM 的注意力权重反向推断每个 token 对最终奖励的贡献。近似方法，无需额外标注。

## 五、算法演进：从 PPO 到 DAPO/GSPO

这是 2026 年新增的高阶考点，反映了偏好优化算法的快速迭代。

### 5.1 PPO → DPO → GRPO → DAPO → GSPO 的演进逻辑

**考察意图：** 出现 5+ 次，考察对算法谱系的系统性理解。

**回答框架：**

这条演进线的核心驱动力是 **简化训练流程 + 提升稳定性** ：

**PPO（2017）：** 完整的 RL 框架——需要 4 个模型、在线采样、GAE 优势估计、多步更新。效果好但工程复杂度高。

**DPO（2023）：** 绕过 RM + RL，直接从偏好数据优化策略。极大简化了流程（只需 1 个模型 + 1 个冻结参考模型），但是离线学习，存在分布偏移问题。

**GRPO（2024）：** 保留 RL 框架但去掉 Critic，用组内归一化替代 GAE。在 PPO 的效果和 DPO 的简洁性之间取得平衡。

**DAPO（2024-2025）：** 在 GRPO 基础上探索更激进的优化——如动态 clip 范围、token 级熵正则化、解耦 KL 约束。追求更好的探索-利用平衡。

**GSPO（Generalized Supervised Policy Optimization）：** 将策略优化重新回归为监督学习框架，但用经过筛选和加权的样本做训练。核心思想是：与其用复杂的 RL 目标函数，不如用"高质量样本 + 权重调整"的方式做加权 SFT。

演进的总趋势：从"完整 RL"到"半 RL 半监督"再到"回归监督学习 + 精细采样"。底层逻辑是：LLM 的对齐不一定需要完整的 RL 框架，关键是能有效利用偏好信号调整模型行为。

### 5.2 在线 RL vs 离线 RL 在 RLHF 中的区别

**考察意图：** 出现 4+ 次，考察对 RLHF 范式的宏观理解。

**回答框架：**

**在线 RL（如标准 PPO/GRPO）：** 策略模型在训练过程中实时生成 response（rollout），由 RM 打分，用打分结果更新策略。"在线"指的是数据分布始终来自当前策略。优势是训练数据与策略分布一致（无分布偏移），劣势是 rollout 成本高（每步都要做 LLM 推理）。

**离线 RL（如标准 DPO）：** 偏好数据在训练前预先收集好，训练过程中不再采样新数据。"离线"指的是数据分布来自收集时的策略（通常是 SFT 模型），与当前正在优化的策略不同。优势是无需在线采样，训练效率高。劣势是分布偏移——随着策略更新，训练数据越来越不能代表当前策略的行为分布。

**迭代式 DPO：** 介于在线和离线之间——每轮 DPO 后用更新后的策略重新采样偏好数据，下一轮用新数据训练。本质上是"多轮离线"逼近"在线"，缓解分布偏移但增加了采样开销。

RLHF 整体属于 **在线与离线的混合范式** ：奖励模型的训练是离线的（用预收集的偏好数据），策略优化阶段（PPO/GRPO）是在线的（实时 rollout + 更新）。

## 六、高频对比总结：PPO vs DPO vs GRPO

这个三方对比在题库中以不同角度出现了 **25+ 次** ，以下是一个覆盖所有常见考察维度的综合对比：

| 维度 | PPO | DPO | GRPO |
| --- | --- | --- | --- |
| **优化范式** | 在线 RL | 离线监督学习 | 在线 RL（简化版） |
| **是否需要 RM** | 是（显式 RM 打分） | 否（隐式建模在策略中） | 是（用于组内归一化） |
| **是否需要 Critic** | 是（Value Model） | 否 | 否 |
| **模型数量** | 4 个 | 2 个（Policy + Ref） | 3 个（Policy + Ref + RM） |
| **优势估计** | GAE（Critic-based） | 无（直接用偏好对的 logit 差） | 组内归一化（Critic-free） |
| **分布偏移风险** | 低（在线采样） | 高（离线数据） | 低（在线采样） |
| **训练复杂度** | 高 | 低 | 中 |
| **显存开销** | 最高 | 最低 | 中等 |
| **Reward Hacking 风险** | 中（受 KL 约束保护） | 低（无显式 RM 可被利用） | 中（仍使用 RM） |
| **适用场景** | 追求最佳对齐效果 | 训练简单、资源有限 | 需要在线学习但想省 Critic |

面试中的关键洞察：DPO 最简单但受限于离线数据；PPO 最强大但工程复杂度最高；GRPO 是两者的折中——保留了在线学习的优势，去掉了最昂贵的 Critic 组件。

**"PPO vs DPO vs GRPO"成为标准化三方对比题。** 这道题出现了 25+ 次，已经是 RLHF 面试的"标配"。面试官期望的不是背诵表格，而是能从第一性原理解释：为什么 DPO 能绕过 RM？GRPO 如何用组内归一化替代 GAE？PPO 的 4 模型架构为什么在工程上最复杂？

**GRPO 从论文走向面试主流。** DeepSeek-R1 的发布使 GRPO 从小众算法变成了面试热点。面试官的考察已经深入到实现细节——rollout 如何复用、组内归一化的 K 值选择、移除 clip 的后果、token 级奖励分配。

**Reward Hacking 防御成为高阶必考题。** 随着 RLHF 在生产中的大规模部署，reward hacking 从理论问题变成了实际工程问题。面试官期望候选人能列举至少 3 种典型表现和对应的防御机制，并能讨论防御策略之间的权衡。
