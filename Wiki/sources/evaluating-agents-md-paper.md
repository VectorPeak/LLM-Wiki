# Evaluating AGENTS.md

## 摘要

这篇论文研究仓库级上下文文件，尤其是 `AGENTS.md` 这类给 coding agent 的项目说明，是否能实际提升 agent 在代码仓库中的表现。它与 [[智能体可读性]] 直接相关：文档不是越多越好，关键在于是否能被 agent 正确利用并转化为行为改进。

## 关键断言

- 为 coding agent 提供仓库级上下文已成为广泛实践。
- 这种实践虽然被工具开发者鼓励，但需要严谨评估。
- 评价重点应落到 agent 是否更好完成任务，而不是文档本身是否看起来完整。
