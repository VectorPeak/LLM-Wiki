# VibeCoding 进化论：OpenSpec

## 摘要

这篇自笔记围绕 OpenSpec 和 SDD 展开，把 Vibe Coding 与规格驱动开发做对比，并整理了 OpenSpec 的基本流程、指令、项目设计原理和面试问题。

## 关键断言

- Vibe Coding 更强调快速生成，SDD 更强调先把规格、设计和任务拆清楚。
- OpenSpec 的价值在于把需求转成项目内可追踪、可执行、可归档的规格文档。
- `/explore`、`/propose`、`/apply`、`/archive` 等指令体现了从探索到落地再到归档的流程。

## 说明

这是使用者自笔记，适合作为面试准备和个人方法论线索，不应直接当成外部事实来源。涉及 OpenSpec 当前能力和命令时，需要再查官方文档。
