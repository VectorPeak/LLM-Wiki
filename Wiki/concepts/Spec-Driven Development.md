# Spec-Driven Development

## 定义

Spec-Driven Development，规格驱动开发，是先把需求、设计、任务和验收边界写清楚，再让 AI 或工程团队按规格实现的开发方式。

## 与 Vibe Coding 的对比

Vibe Coding 像“边聊边施工”，反馈快但容易跑偏；SDD 像“先画图纸再施工”，前期更重，但更可复查、可交接、可追责。

## 对 AI 编程的价值

SDD 将隐含意图变成项目内可读工件，这与 [[智能体可读性]] 高度一致。对 agent 来说，规格不是形式主义，而是减少误解和漂移的外部状态。
