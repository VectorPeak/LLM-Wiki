# 智能体可读性

## 定义

智能体可读性是指一个系统能否被 AI agent 直接发现、理解、验证和修改。对 Codex 这类编码智能体来说，不在本地仓库、文档、脚本、日志、测试或工具输出中的信息，运行时几乎等同于不存在。

## 与人类可读性的区别

人类可读性强调解释性、叙事和背景感；智能体可读性更强调可导航、可枚举、可解析、可验证和可执行。一个长篇手册可能对人类有用，但对 agent 来说可能挤占上下文并造成规则漂移。

## 典型做法

- 短 `AGENTS.md` 作为地图，而不是百科全书。
- 结构化 `docs/`、设计文档、执行计划和质量报告。
- 通过 lint、测试和 CI 强制架构不变量。
- 让 UI、日志、指标和追踪对 agent 可查询。
- 把人类审查意见沉淀为规则、文档或自动化检查。

## 对 LLMWiki 的启发

LLMWiki 的 `Schema`、`Wiki/index.md`、`Wiki/hot.md` 和 `Wiki/log.md` 本质上是在提升知识库的智能体可读性。它们不是装饰性文档，而是让后续 Codex 能够继续维护知识图谱的导航和控制层。
